Metamask Extension Bug Swaps Changing The Destination Token Cha

Symptom

  • When attempting to swap tokens using the Metamask extension, users may encounter an issue where changing the destination token unexpectedly alters the source token instead, leading to unintended transaction configurations.

Technical Root Cause

  • This error occurs due to a bug within the Metamask extension's swap functionality, where the token selection inputs are not properly isolated, causing changes to one input to inadvertently affect the other.

Manual Overrides

  • To temporarily bypass this issue, users can try the following manual overrides:
  • Directly editing the transaction data in the extension's advanced settings
  • Utilizing a third-party swapping service that doesn't rely on the Metamask extension for token selection
  • Employing a custom script to interact with the Ethereum blockchain, such as: json { "from": "0x...source_address", "to": "0x...destination_address", "value": "1.0 ether", "gas": "20000", "gasPrice": "20.0 ether" } Consider using an external console for more complex transaction configurations.
