Metamask Extension Bug Swaps Changing The Destination Token Cha

Symptom

  • When attempting to swap tokens using the Metamask extension, changing the destination token unexpectedly alters the source token instead of updating the destination token.

Technical Root Cause

  • The issue arises from a bug in the Metamask extension's swap functionality, where the logic for updating the destination token is incorrectly linked to the source token selection mechanism.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Refresh the Metamask extension and retry the swap operation.
  • Reset the Metamask extension's token list by deleting the tokenList.json file.
  • Update the Metamask extension to the latest version and ensure the Ethereum network is properly configured, as shown in the following code block:
{
  "network": "ethereum",
  "chainId": 1,
  "tokenList": []
}

Consider using an external console to verify the Ethereum network configuration.

Prerequisites

  • Metamask extension installed and configured.
  • Ethereum network selected in the Metamask extension.
  • Latest version of the Metamask extension updated.
