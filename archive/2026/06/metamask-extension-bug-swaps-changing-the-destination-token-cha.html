Metamask Extension Bug Swaps Changing The Destination Token Cha

Symptom

  • When attempting to swap tokens using Metamask, changing the destination token unexpectedly alters the source token instead of updating the destination token as intended.

Technical Root Cause

  • This error occurs due to a bug within the Metamask extension, specifically affecting the swap functionality. The issue arises from a mismanaged state update, where the source token is incorrectly updated when the destination token is changed.

Manual Overrides

  • To resolve this issue, users can attempt the following manual overrides:
  • Refresh the Metamask extension and retry the swap operation.
  • Manually input the destination token's contract address to bypass the token selection interface.
  • Utilize the following JSON configuration to force-update the token list:
{
  "tokens": [
    {
      "address": "0x...destination_token_address",
      "symbol": "DEST",
      "decimals": 18

  ]
}
