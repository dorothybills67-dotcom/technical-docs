Symptom

Technical Root Cause

  • The error occurs due to asynchronous storage writes not completing before the extension proceeds with other operations, causing data inconsistency or loss.

Manual Overrides

  • Use the mm.eth.clearPendingTransactions() function to clear any pending transactions that may be causing the delay.
  • Implement a retry mechanism using async/await to ensure storage writes are completed before proceeding.
  • Utilize the eth_waitUntil function to pause execution until the storage writes are finished, as shown in the following code block:
eth_waitUntil(() => {
  return mm.eth.getPendingTransactions().length === 0;
