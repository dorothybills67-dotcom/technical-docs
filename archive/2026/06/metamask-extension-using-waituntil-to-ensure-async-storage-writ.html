Metamask Extension Using WaitUntil To Ensure Async Storage Writes Finish

Symptom

  • The Metamask extension is stuck on the "Using waitUntil to ensure async storage writes finish" message, indicating a potential issue with asynchronous storage writes.

Technical Root Cause

  • This error occurs when the Metamask extension is unable to complete asynchronous storage writes, causing the application to wait indefinitely for the writes to finish.

Manual Overrides

  • To resolve this issue, try the following manual overrides:
  • Check the Ethereum network connection and ensure it is stable.
  • Verify that the Metamask extension is up-to-date and restart the browser if necessary.
  • Use the following code to manually trigger a storage write:
{
  "method": "eth_accounts",
  "params": [],
  "id": 1
}

Consider using an external console to execute the above command for more control over the process.

Prerequisites

  • A stable Ethereum network connection
  • The latest version of the Metamask extension
  • Basic knowledge of JSON-RPC commands
