Metamask Extension Using WaitUntil To Ensure Async Storage Writes Finish

Symptom

  • The user encounters an issue where the Metamask extension is using waitUntil to ensure async storage writes finish, resulting in a delayed or unresponsive browser experience.

Technical Root Cause

  • This error occurs due to the asynchronous nature of storage writes in the Metamask extension, where waitUntil is used to pause the execution of the code until the storage writes are complete, causing a potential delay.

Manual Overrides

  • To resolve this issue, try the following manual overrides:
  • Increase the timeout value for waitUntil to allow for more time to complete the storage writes.
  • Implement a retry mechanism to handle failed storage writes.
  • Use a more efficient storage solution, such as a caching layer, to reduce the load on the async storage.
