Metamask Extension Using Waituntil To Ensure Async Storage Writ

Symptom

  • Users encounter an issue where the Metamask extension is using waitUntil to ensure async storage writes finish, resulting in delayed or unresponsive transactions on the Ethereum (ETH) network.

Technical Root Cause

  • This error occurs due to the asynchronous nature of storage writes in the Metamask extension, which can lead to timing issues and delays in transaction processing.

Manual Overrides

  • To resolve this issue, users can try the following manual overrides:
  • Increase the timeout value for async storage writes
  • Implement a retry mechanism for failed transactions
  • Use a more efficient storage solution, such as a caching layer
