Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users experience a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and potential loss of funds.

Technical Root Cause

  • The error occurs due to a prolonged loop in the transfer process, often caused by network congestion, insufficient gas, or incorrect smart contract configuration.

Manual Overrides

  • Increase the gas limit for the transaction to ensure sufficient resources are allocated:
  • Modify the transaction parameters to include a higher gas limit
  • Utilize a different transfer method to bypass the loop
  • Run the following command to override the default gas limit:
tonos-cli call <contract_address> transferJetton '{"dst":"<destination_address>","amount":<amount>,"gas":1000000}' --abi <abi_file>

Consider utilizing an external console for more advanced troubleshooting and configuration options.
