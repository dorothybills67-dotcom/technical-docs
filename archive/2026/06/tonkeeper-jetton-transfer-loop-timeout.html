Symptom

  • The user encounters a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons, indicating a failure in the transaction process due to a timeout in the transfer loop.

Technical Root Cause

  • The error occurs due to the Tonkeeper wallet's inability to complete the jetton transfer within the allotted time frame, often caused by network congestion, high gas fees, or issues with the smart contract.

Manual Overrides

  • Check the current network status and gas prices to determine if high fees are causing the timeout.
  • Adjust the transaction settings to prioritize speed over cost, if feasible.
  • Utilize the following command to override default settings:
tonos-cli call <wallet_address> submitTransaction "<transaction_data>" --timeout 60

Consider utilizing an external console for more advanced debugging and manual transaction management.

Prerequisites

  • Familiarity with the TON blockchain and its ecosystem
  • Understanding of smart contract interactions and gas fees
  • Access to the tonos-cli command-line tool
