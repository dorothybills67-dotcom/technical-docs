Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users experience a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an inability to complete the transfer process.

Technical Root Cause

  • The error occurs due to a prolonged loop in the transfer process, often caused by network congestion, insufficient gas, or incorrect configuration of the smart contract.

Manual Overrides

  • To resolve the issue, try the following:
  • Increase the gas limit for the transaction
  • Verify the smart contract configuration and ensure it is correctly deployed
  • Use a custom script to bypass the loop and force-complete the transaction, such as:
tonos-cli call <contract_address> transfer '{"dest":"<destination_address>","amount":<amount>}' --abi <abi_file>

Consider utilizing an external console for more complex debugging and transaction management.

Prerequisites

  • Tonos-cli installed and configured
  • ABI file for the smart contract
  • Sufficient balance and gas for transactions
