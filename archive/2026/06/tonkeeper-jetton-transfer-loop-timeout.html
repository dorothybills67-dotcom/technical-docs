Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • The user encounters an error message indicating a timeout during a Tonkeeper Jetton transfer, preventing the completion of the transaction. The error message explicitly states "Tonkeeper Jetton transfer loop timeout," and the transfer process appears to be stuck in an infinite loop.

Technical Root Cause

  • The error occurs due to a misconfiguration in the Tonkeeper settings or a temporary issue with the TON network, causing the transfer loop to exceed the maximum allowed time, resulting in a timeout.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Check the Tonkeeper configuration files for any syntax errors or incorrect settings.
  • Use the following JSON configuration to override the default timeout settings:
{
  "timeout": 30000,
  "retry_count": 5
}
  • Restart the Tonkeeper application to apply the changes. For more advanced troubleshooting, consider using an external console to execute low-level commands.

Prerequisites

  • Tonkeeper application installed and configured
  • Basic understanding of TON network and Tonkeeper settings
  • Access to the Tonkeeper configuration files
