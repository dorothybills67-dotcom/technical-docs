Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users experience a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an inability to complete the transfer process.

Technical Root Cause

  • The error occurs due to a synchronization issue between the Tonkeeper wallet and the TON blockchain, causing the transfer loop to exceed the maximum allowed time, leading to a timeout.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Increase the transfer timeout threshold using the tonos-cli command
  • Reset the Tonkeeper wallet settings to default values
  • Use a custom configuration file to override the default transfer settings, as shown in the following code block:
{
  "transfer": {
    "timeout": 30000
  }
}

Consider using an external console to execute these commands for better control.

Prerequisites

  • Tonos-cli installed and configured on the system
  • A basic understanding of TON blockchain and Tonkeeper wallet functionality
  • An active internet connection to synchronize with the TON blockchain
