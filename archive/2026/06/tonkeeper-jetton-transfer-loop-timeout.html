Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users encounter a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an inability to complete the transfer process.

Technical Root Cause

  • The error occurs due to a prolonged loop in the transfer process, often caused by network congestion, high gas fees, or issues with the smart contract governing the Jetton transfer.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Increase the gas limit to prioritize the transaction
  • Reduce the transfer amount to minimize network load
  • Use a custom transaction configuration, such as:
{
  "sender": "user_address",
  "receiver": "recipient_address",
  "amount": 10,
  "gas": 20000
}

Consider utilizing an external console for more advanced troubleshooting and configuration options.

Prerequisites

  • Basic understanding of TON blockchain and Tonkeeper wallet
  • Familiarity with smart contract interactions
  • Access to the Tonkeeper wallet and transaction history
