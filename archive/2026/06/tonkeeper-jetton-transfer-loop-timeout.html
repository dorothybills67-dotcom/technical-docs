Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users experience a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an inability to complete the transfer process.

Technical Root Cause

  • The error occurs due to a temporary mismatch between the Tonkeeper wallet's internal state and the TON blockchain's current state, causing the transfer loop to exceed the maximum allowed time.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Increase the transfer timeout threshold using the tonos-cli command
  • Verify the wallet's internal state by checking the jetton_balance using the following code block: bash tonos-cli getjettonbalance <jetton_address> --wallet <wallet_address>
  • Restart the Tonkeeper wallet application to refresh its internal state For a more comprehensive solution, consider utilizing an external console to execute the above commands, which may provide more detailed output and better control over the process.

Prerequisites

  • Tonos-cli installed and configured on the system
  • Tonkeeper wallet application installed and set up
  • Basic understanding of TON blockchain and Tonkeeper wallet functionality
