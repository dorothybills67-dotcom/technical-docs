Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons within the TON ecosystem, indicating a failure in the transaction process.

Technical Root Cause

  • The error occurs due to a timeout in the transfer loop, which can be caused by network congestion, high gas prices, or issues with the smart contract governing the jetton transfer.

Manual Overrides

  • To resolve the issue, try the following:
  • Increase the gas limit for the transaction to prioritize its processing.
  • Verify that the sender's wallet has sufficient funds to cover the transaction fees.
  • Adjust the transaction timeout parameters using the following code:
{
  "timeout": 300, // example timeout in seconds
  "gas_limit": 2000000 // example gas limit
}

Consider using an external console for more advanced troubleshooting and configuration adjustments.

Prerequisites

  • Familiarity with TON ecosystem and its tools
  • Understanding of smart contract interactions
  • Access to the Tonkeeper wallet and its configuration settings
