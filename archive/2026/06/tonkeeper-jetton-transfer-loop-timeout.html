Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • The user encounters a timeout error when attempting to transfer Tonkeeper Jetton, resulting in a failed transaction. The error message "Tonkeeper Jetton transfer loop timeout" is displayed, indicating that the transfer process has exceeded the allowed time limit.

Technical Root Cause

  • The error occurs due to a prolonged delay in the transaction processing, often caused by network congestion, high gas fees, or issues with the Tonkeeper smart contract.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Increase the gas limit to prioritize the transaction
  • Reduce the transaction amount to minimize network load
  • Use a custom RPC endpoint with a faster node
