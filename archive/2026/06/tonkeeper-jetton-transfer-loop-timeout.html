Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users experience a timeout error when attempting to transfer Tonkeeper Jettons, resulting in a failed transaction and an error message indicating a transfer loop timeout.

Technical Root Cause

  • The error occurs due to a temporary mismatch between the Tonkeeper wallet's internal state and the TON blockchain, causing the transfer loop to exceed the maximum allowed time, resulting in a timeout.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Increase the timeout threshold using the --timeout flag
  • Reset the Tonkeeper wallet's internal state using the tonkeeper reset command
  • Update the wallet's configuration using the following JSON code block:
{
  "timeout": 30000,
  "retry_count": 5
}

Prerequisites: * Tonkeeper wallet version 1.2.0 or higher * TON blockchain node synced with the latest block * Sufficient TON coins for transaction fees

Consider utilizing an external console, such as tonos-cli, to execute these commands.
