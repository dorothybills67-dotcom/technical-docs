Symptom

  • The user encounters a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons, indicating a failure in the transaction process due to a timeout in the loop that handles the transfer.

Technical Root Cause

  • The error occurs due to the Tonkeeper wallet's jetton transfer mechanism exceeding the predefined timeout threshold, likely caused by network congestion, slow transaction processing, or issues with the smart contract governing the jetton transfer.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Increase the timeout threshold in the Tonkeeper wallet settings to allow for more time to process the transaction.
  • Use the tonos-cli command to manually invoke the transfer function with an increased timeout: bash tonos-cli call <jetton_contract_address> transfer "{\"dest\":\"<recipient_address>\",\"value\":\"<amount>\"}" --timeout 60
  • Check the Ton blockchain for any ongoing issues that might be affecting transaction processing times.

Prerequisites: - Tonos-cli installed and configured on your system - Basic understanding of TON blockchain and smart contracts - Access to the Tonkeeper wallet and its settings

Consider utilizing an external console for more complex debugging and troubleshooting steps.
