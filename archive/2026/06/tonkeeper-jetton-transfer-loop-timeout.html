Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons within the TON ecosystem, resulting in failed transactions and potential loss of funds.

Technical Root Cause

  • The error occurs due to a timeout in the Tonkeeper protocol, which is responsible for facilitating jetton transfers. This timeout can be caused by network congestion, high transaction volumes, or issues with the Tonkeeper smart contract.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Increase the transaction timeout threshold
  • Implement a retry mechanism for failed transactions
  • Update the Tonkeeper smart contract to optimize transaction processing
