Symptom

  • The user encounters a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons, indicating a failure in the transaction process due to a timeout in the loop that handles the transfer.

Technical Root Cause

  • The error occurs due to a prolonged delay in the transaction confirmation process, exceeding the predefined timeout threshold set by the Tonkeeper system for jetton transfers, which is crucial for maintaining the integrity and efficiency of transactions within the TON ecosystem.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Increase the timeout threshold in the Tonkeeper settings to allow for more time for transaction confirmation.
  • Check the blockchain for any congestion that might be causing delays in transaction processing.
  • Utilize the following command to manually reset the transfer loop:
tonos-cli call -6 <transfer_contract_address> reset_transfer_loop {}

Consider utilizing an external console for advanced troubleshooting and diagnostics.

Prerequisites

  • A basic understanding of the TON blockchain and its ecosystem.
  • Familiarity with the Tonkeeper interface and its settings.
  • Access to the tonos-cli command-line tool for interacting with the TON blockchain.
