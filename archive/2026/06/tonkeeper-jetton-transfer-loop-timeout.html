Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons within the TON ecosystem, resulting in failed transactions and potential loss of funds.

Technical Root Cause

  • The error occurs due to a timing mismatch between the Tonkeeper wallet and the TON blockchain, causing the transfer loop to exceed the maximum allowed timeout period.

Manual Overrides

  • Increase the timeout threshold manually using the Tonkeeper API.
  • Verify the wallet's clock synchronization with the TON blockchain to ensure accurate timing.
  • Utilize the following JSON configuration to adjust the transfer loop settings:
{
  "timeout": 30000,
  "retry_count": 5,
  "transfer_loop_delay": 5000
}

Consider utilizing an external console for advanced troubleshooting and configuration adjustments.

Prerequisites

  • Tonkeeper wallet installed and configured correctly
  • Basic understanding of TON blockchain and jetton transfer mechanics
  • Access to the Tonkeeper API documentation and developer tools
