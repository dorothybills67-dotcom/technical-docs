Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons, resulting in failed transactions and potential losses.

Technical Root Cause

  • The error occurs due to a timeout in the Tonkeeper Jetton transfer loop, which can be caused by network congestion, high gas fees, or issues with the Tonkeeper smart contract.

Manual Overrides

  • Increase the gas limit for the transaction to prioritize it in the network:
  • Update the transaction settings to use a higher gas price
  • Use a custom transaction sender to manually set the gas limit
  • Implement a retry mechanism using the following code:
while ! tonos-cli call $ADDR sendJettons \
  --input "{\"dest\":\"$DEST\",\"amount\":\"$AMOUNT\"}" \
  --abi JettonWallet.abi.json; do
  sleep 1
done

Consider using an external console for more advanced debugging and transaction management.

Prerequisites

  • Tonos-cli installed and configured on the system
  • Jetton Wallet ABI file (JettonWallet.abi.json) available
  • Sufficient funds and jettons in the wallet for the transaction
