Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons within the TON ecosystem, resulting in failed transactions and potential loss of funds.

Technical Root Cause

  • The error occurs due to a timeout in the Tonkeeper's jetton transfer loop, which is likely caused by network congestion, high gas fees, or incorrect configuration of the transfer parameters.

Manual Overrides

  • Increase the gas limit to accommodate for the increased network load:
  • Increase the --gas parameter when initiating the transfer
  • Adjust the --gas-price to a higher value to prioritize the transaction
  • Use the following code to override the default transfer settings:
tonos-cli call <wallet_address> override_transfer_config '{"gas": 100000, "gas_price": 1000000000}' --abi Wallet.abi.json

Consider utilizing an external console for more advanced troubleshooting and configuration options.

Prerequisites

  • Familiarity with the TON command-line interface (CLI)
  • Access to the tonos-cli tool
  • Understanding of the Tonkeeper wallet configuration and settings
