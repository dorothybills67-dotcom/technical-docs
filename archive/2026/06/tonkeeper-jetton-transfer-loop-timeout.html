Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users encounter a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and potential loss of funds. The error message "Tonkeeper Jetton transfer loop timeout" is displayed, indicating a failure in the transfer process.

Technical Root Cause

  • The error occurs due to a loop timeout in the Tonkeeper smart contract, which is responsible for handling Jetton transfers. This loop timeout can be caused by various factors, including network congestion, contract errors, or insufficient gas.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Increase the gas limit for the transaction to ensure sufficient resources for the contract execution.
  • Verify the contract code for any errors or bugs that may be causing the loop timeout.
  • Use the following JSON-RPC command to override the contract settings:
{
  "method": "ton_sendTransaction",
  "params": {
    "contract": "JettonContract",
    "function": "transfer",
    "args": ["recipient_address", "amount"],
    "gas": "1000000"
  }
}

Consider using an external console for more advanced troubleshooting and contract interaction.

Prerequisites

  • Ton Wallet installed and configured
  • Sufficient TON funds for gas and transaction fees
  • Basic understanding of Tonkeeper smart contract functionality
