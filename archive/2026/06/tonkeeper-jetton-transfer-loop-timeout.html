Tonkeeper Jetton Transfer Loop Timeout

Symptom

  • Users encounter a timeout error when attempting to transfer Tonkeeper Jettons, resulting in failed transactions and an unclear status of their jetton balance.

Technical Root Cause

  • The error occurs due to a temporary mismatch between the user's wallet state and the blockchain's state, causing the transfer loop to exceed the allowable timeout threshold.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Increase the timeout threshold using the tonos-cli command
  • Restart the Tonkeeper wallet application to resynchronize with the blockchain
  • Utilize the following code block to manually update the wallet's state:
{
  "type": "update",
  "params": {
    "wallet_address": "user_wallet_address",
    "jetton_balance": "updated_jetton_balance"
  }
}

Consider utilizing an external console for more complex troubleshooting scenarios.

Prerequisites

  • Familiarity with the TON blockchain and its ecosystem
  • Access to the tonos-cli command-line tool
  • A basic understanding of JSON formatting and syntax
