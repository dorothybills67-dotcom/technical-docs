Symptom

  • Users experience a "Tonkeeper Jetton transfer loop timeout" error when attempting to transfer jettons within the TON ecosystem, resulting in failed transactions and potential losses.

Technical Root Cause

  • The error occurs due to a timeout in the transfer loop, which can be caused by network congestion, incorrect configuration, or issues with the Tonkeeper wallet.

Manual Overrides

  • To resolve the issue, try the following:
  • Check the TON network status for any ongoing maintenance or issues
  • Verify that the Tonkeeper wallet is updated to the latest version
  • Use the following command to reset the transfer loop:
tonos-cli call <wallet_address> resetTransferLoop {} --abi TonkeeperWallet.abi

Consider utilizing a dedicated console for troubleshooting to avoid any potential conflicts with your primary environment.

Prerequisites

  • TON ecosystem account
  • Tonkeeper wallet installed and configured
  • Basic understanding of TON network and Tonkeeper functionality
