Tronlink Bandwidth Error

Symptom

  • Users encounter an error message indicating a bandwidth issue when attempting to perform transactions or interact with smart contracts on the TRON network using TronLink.

Technical Root Cause

  • The error occurs due to insufficient bandwidth, which is a resource required for transactions on the TRON network. Each account has a limited amount of bandwidth that replenishes over time. When this limit is exceeded, transactions cannot be processed.

Manual Overrides

  • Increase the account's bandwidth by freezing TRX tokens, which allocates more bandwidth.
  • Adjust the transaction parameters to require less bandwidth, such as reducing the transaction size or waiting for bandwidth to replenish.
  • Use the following JSON-RPC command to check the current bandwidth:
{
  "jsonrpc": "2.0",
  "method": "tron_getAccount",
  "params": [
    {
      "address": "your_tron_address"
    }
  ],
  "id": 1
}

Consider utilizing an external console for more complex debugging and direct access to the TRON network.

Prerequisites

  • A TRON account with TronLink installed and set up.
  • Basic understanding of TRON network resources (bandwidth and energy).
  • Familiarity with JSON-RPC commands for interacting with the TRON network.
