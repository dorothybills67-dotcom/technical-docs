Symptom

  • Users encounter a "TronLink bandwidth error" when attempting to perform transactions or interact with smart contracts on the TRON network, resulting in failed or pending transactions.

Technical Root Cause

  • The error occurs due to insufficient bandwidth or energy on the user's TRON account, which is required to execute transactions on the network. This is a fundamental aspect of the TRON protocol, where bandwidth and energy are used to measure the resources consumed by transactions.

Manual Overrides

  • Increase the account's bandwidth by freezing TRX tokens for a specified period.
  • Adjust the transaction parameters, such as the transaction timeout and confirmation threshold.
  • Use the following JSON-RPC command to query the account's bandwidth:
{
  "jsonrpc": "2.0",
  "method": "tron_getAccount",
  "params": [
    {
      "address": "your_tron_address"
    }
  ],
  "id": 1
}

Consider utilizing an external console to execute these commands for a more streamlined experience.

Prerequisites

  • A TRON account with a valid address
  • A basic understanding of TRON's bandwidth and energy mechanisms
  • The TRONLink extension or a compatible wallet installed
