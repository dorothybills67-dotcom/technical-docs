Symptom

  • Users experience a "TronLink bandwidth error" when attempting to interact with the TRON blockchain, resulting in failed transactions and inability to perform actions on the network.

Technical Root Cause

  • The error occurs due to insufficient bandwidth on the user's TRON account, which is required to process transactions and interact with smart contracts on the network.

Manual Overrides

  • Increase the account's bandwidth by freezing TRX tokens, which allocates more bandwidth to the account.
  • Use a different wallet or account with sufficient bandwidth to perform the desired actions.
  • Adjust the transaction parameters to require less bandwidth, such as reducing the transaction size or using a different transaction type, as shown in the following example:
{
  "from": "TXaFZzNMxhg7KFuQhTj7oq6xVf2xwbz9rE",
  "to": "TYMxhX25bpwX7xSBLxkbVfJHn9zZrLjk2s",
  "value": 100,
  "bandwidthLimit": 10000
}

Consider using an external console for more complex transactions.

Prerequisites

  • A TRON account with a positive balance
  • The TronLink extension installed and configured
  • A basic understanding of TRON's bandwidth and transaction mechanics
