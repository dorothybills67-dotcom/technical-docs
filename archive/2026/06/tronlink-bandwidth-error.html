TronLink Bandwidth Error

Symptom

  • Users encounter an error message indicating a bandwidth issue when attempting to perform transactions or interact with smart contracts on the TRON network using TronLink.

Technical Root Cause

  • The error occurs due to insufficient bandwidth, which is a resource required for transaction processing on the TRON network. Bandwidth is typically allocated based on the number of TRX (TRON's native cryptocurrency) held and frozen by a user.

Manual Overrides

  • Increase the bandwidth by freezing more TRX.
  • Adjust the transaction parameters to require less bandwidth.
  • Utilize the tronweb library to manually set the bandwidth:
{
  "transaction": {
    "from": "your_account",
    "to": "recipient_account",
    "value": 100,
    "bandwidth": 10000
  }
}

Consider utilizing an external console for more complex transactions.

Prerequisites

  • TRX frozen for bandwidth allocation
  • TronLink wallet installed and configured
  • Basic understanding of TRON network transaction mechanics
