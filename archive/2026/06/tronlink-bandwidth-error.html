Tronlink Bandwidth Error

Symptom

  • Users encounter an error message indicating a TronLink bandwidth error when attempting to perform transactions or interact with the TRON ecosystem through TronLink.

Technical Root Cause

  • The error occurs due to insufficient bandwidth on the user's TRON account, which is required for transaction processing and smart contract execution.

Manual Overrides

  • Increase the account's bandwidth by freezing TRX tokens for bandwidth:
  • Freeze a minimum of 144 TRX to obtain a bandwidth of 1440 bytes per day
  • Use the TronLink wallet interface to freeze and unfreeze TRX for bandwidth
  • Configure the following JSON to adjust bandwidth parameters:
{
  "address": "user_tron_address",
  "frozen_balance": {
    "frozen_amount": 144000000,
    "resource": "BANDWIDTH"
  }
}

Consider utilizing an external console for more granular control over transaction and bandwidth management.

Prerequisites

  • TRX tokens available for freezing
  • TronLink wallet installation and setup
  • Basic understanding of TRON ecosystem and transaction processing
