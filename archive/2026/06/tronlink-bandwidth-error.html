Tronlink Bandwidth Error

Symptom

  • Users encounter an error message indicating a bandwidth issue when attempting to perform transactions or interact with smart contracts on the Tron network using TronLink.

Technical Root Cause

  • The error occurs due to insufficient bandwidth, which is a resource required for transactions on the Tron network. Each account has a limited amount of bandwidth that can be used within a certain time frame.

Manual Overrides

  • Increase the account's bandwidth by freezing TRX tokens, which in turn increases the account's bandwidth.
  • Use the TronLink interface to adjust transaction settings and prioritize bandwidth allocation.
  • Utilize the following JSON payload to update the account's bandwidth parameters:
{
  "visible": true,
  "permission": {
    "type": 2,
    "threshold": 1,
    "keys": [
      {
        "address": "your_address",
        "weight": 1
      }
    ]
  },
  "owner_address": "your_address",
  "active": {
    "threshold": 1,
    "keys": [
      {
        "address": "your_address",
        "weight": 1
      }
    ]
  }
}

Consider using an external console for more advanced configuration options.

Prerequisites

  • TronLink extension installed and configured
  • TRX tokens available for freezing
  • Basic understanding of Tron network resources (bandwidth and energy)
