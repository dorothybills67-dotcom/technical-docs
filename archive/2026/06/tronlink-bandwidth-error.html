Symptom

  • Users encounter a "TronLink bandwidth error" when attempting to perform transactions or interact with smart contracts on the TRON network, resulting in failed or pending transactions.

Technical Root Cause

  • The error occurs due to insufficient bandwidth, which is a resource required for transaction processing on the TRON network. Each account has a limited amount of bandwidth, and when this limit is exceeded, transactions cannot be processed.

Manual Overrides

  • Increase the account's bandwidth by freezing TRX tokens, which allocates more bandwidth to the account.
  • Use a different account with sufficient available bandwidth to perform the desired transactions.
  • Temporarily adjust the transaction parameters to reduce the bandwidth required, such as by lowering the transaction fee:
tronweb.transactionBuilder.sendTrx(
  "recipient_address", 
  100, 
  "sender_address"
)

Consider using an external console for more complex bandwidth management tasks.

Prerequisites

  • TRON wallet with sufficient TRX balance
  • TRON network connection
  • TronLink extension or compatible wallet software
