Symptom

  • Users encounter a "TronLink bandwidth error" when attempting to perform transactions or interact with smart contracts on the TRON network using TronLink, a popular wallet and browser extension.

Technical Root Cause

  • The error occurs when the user's account bandwidth is depleted, preventing further transactions or actions on the network. Bandwidth in TRON is a resource that measures the amount of data a user can send to the network within a specific time frame.

Manual Overrides

  • Increase the account bandwidth by freezing TRX tokens, which allocates more bandwidth to the user's account.
  • Check for any pending or stalled transactions that may be consuming bandwidth and cancel them if necessary.
  • Utilize the following JSON-RPC command to query the user's current bandwidth:
{
  "jsonrpc": "2.0",
  "method": "getaccount",
  "params": [
    {
      "address": "user_tron_address"
    }
  ],
  "id": 1
}

Consider utilizing an external console for more advanced debugging and customization options.

Prerequisites

  • TRON wallet with sufficient TRX balance
  • TronLink extension installed and configured
  • Basic understanding of TRON's bandwidth mechanism
