Symptom

  • Users encounter a "TronLink bandwidth error" when attempting to perform transactions or interact with smart contracts on the TRON network, resulting in failed or pending transactions.

Technical Root Cause

  • The error occurs due to insufficient bandwidth or energy on the user's TRON account, which is required to process transactions and execute smart contracts on the network.

Manual Overrides

  • Increase the account's bandwidth by freezing TRX tokens for a specified period.
  • Adjust the transaction parameters, such as the transaction fee or bandwidth limit, to optimize resource utilization.
  • Utilize the following code to check the account's current bandwidth:
curl -X POST \
  https://api.trongrid.io/wallet/getaccount \
  -H 'Content-Type: application/json' \
  -d '{"address": "your_tron_address"}'

Consider utilizing an external console to execute these commands for more flexibility.

Prerequisites

  • A TRON account with a sufficient balance of TRX tokens.
  • The TronLink wallet extension installed and configured.
  • A basic understanding of TRON's resource management system, including bandwidth and energy.
