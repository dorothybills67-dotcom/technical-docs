Symptom

  • Users encounter a "TronLink bandwidth error" when attempting to perform transactions or interact with smart contracts on the TRON network, resulting in failed or pending transactions.

Technical Root Cause

  • The error occurs due to insufficient bandwidth or bandwidth points on the user's TRON account, which are required to process transactions on the network. This can be caused by recent network congestion, insufficient TRX holdings, or incorrect configuration of bandwidth settings.

Manual Overrides

  • Check TRX balance and ensure it is sufficient to cover transaction fees.
  • Adjust bandwidth settings to allocate more bandwidth points to the account.
  • Use the following command to manually override bandwidth settings:
tronweb.transactionBuilder.freezeBalance(tronweb.toSun(10), 3);

Consider utilizing an external console for more advanced troubleshooting and configuration options.

Prerequisites

  • A TRON account with a sufficient TRX balance.
  • The latest version of TronLink installed.
  • A basic understanding of TRON network concepts and transaction processing.
