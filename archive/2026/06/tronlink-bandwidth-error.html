Tronlink Bandwidth Error

Symptom

  • Users encounter an error message indicating a TronLink bandwidth issue, preventing them from completing transactions on the TRON network. The error message may appear when attempting to send TRX, tokens, or interact with smart contracts.

Technical Root Cause

  • The error occurs due to insufficient bandwidth, which is required for transactions on the TRON network. Bandwidth is a resource that is depleted when users perform transactions, and it can be replenished over time or by freezing TRX.

Manual Overrides

  • To resolve the issue, users can try the following:
  • Freeze a portion of their TRX to acquire more bandwidth
  • Wait for a period of time to allow bandwidth to replenish naturally
  • Utilize the following JSON-RPC command to check available bandwidth:
{
  "jsonrpc": "2.0",
  "method": "getaccountbyaddress",
  "params": [
    "user_address"
  ],
  "id": 1
}

For more advanced troubleshooting, consider accessing the network via an external console.

Prerequisites

  • A TRON wallet with a sufficient TRX balance
  • A basic understanding of the TRON network and its resources
  • The TRONLink extension installed and configured in the browser
