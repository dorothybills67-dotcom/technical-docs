Tronlink Bandwidth Error

Symptom

  • Users encounter an error message indicating a TronLink bandwidth issue, preventing them from completing transactions on the TRON network. The error is typically displayed when attempting to send TRX or tokens, and it may also appear as a warning when trying to interact with smart contracts.

Technical Root Cause

  • The error occurs due to insufficient bandwidth on the user's TRON account, which is required to process transactions on the network. Bandwidth is a resource that is consumed by transactions and is replenished over time.

Manual Overrides

  • To resolve the issue, users can try the following:
  • Check their account's available bandwidth and wait for it to replenish.
  • Increase their account's bandwidth by freezing TRX, which will allocate more bandwidth to their account.
  • Use the following JSON-RPC command to check their account's bandwidth:
{
  "jsonrpc": "2.0",
  "method": "getaccount",
  "params": [
    {
      "address": "user's TRON address"
    }
  ],
  "id": 1
}

For more advanced troubleshooting, consider using an external console to execute custom queries.

Prerequisites

  • A TRON account with TRX balance
  • TRON network connection
  • TRONLink wallet installed
