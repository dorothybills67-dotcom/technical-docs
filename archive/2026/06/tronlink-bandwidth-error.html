Tronlink Bandwidth Error

Symptom

  • Users encounter an error message stating "Tronlink bandwidth error" when attempting to perform transactions or interact with smart contracts on the TRON network using TronLink.

Technical Root Cause

  • The error occurs due to insufficient bandwidth or energy on the user's TRON account, which is required to process transactions and execute smart contract functions.

Manual Overrides

  • Increase the account's bandwidth by freezing TRX tokens for a specified period.
  • Adjust the transaction parameters to consume less bandwidth, such as reducing the transaction size or gas limit.
  • Utilize the following JSON-RPC command to estimate the required bandwidth for a transaction:
{
  "jsonrpc": "2.0",
  "method": "tron_estimateBandwidth",
  "params": [
    {
      "from": "your_account_address",
      "to": "recipient_address",
      "value": 0,
      "data": "transaction_data"
    }
  ],
  "id": 1
}

Consider using an external console to execute JSON-RPC commands for more flexibility.

Prerequisites

  • A TRON account with a sufficient TRX balance
  • TronLink installed and configured
  • Basic understanding of TRON's bandwidth and energy mechanisms
