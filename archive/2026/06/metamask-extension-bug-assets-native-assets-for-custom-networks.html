Metamask Extension Bug Assets Native Assets For Custom Networks

Symptom

  • Users are unable to view native assets for custom networks when selected in the Metamask extension, resulting in an incomplete asset list.

Technical Root Cause

  • The issue arises from a bug in the Metamask extension's asset handling mechanism, specifically with regards to custom networks, where native assets are not properly retrieved or displayed.

Manual Overrides

  • To temporarily resolve the issue, users can attempt the following manual overrides:
  • Update the Metamask extension to the latest version
  • Reset the Metamask extension settings to default
  • Utilize the following JSON configuration to manually add native assets for custom networks:
{
  "network": "custom-network",
  "nativeAsset": "native-asset-symbol"
}

Consider utilizing an external console to execute these commands for a more streamlined experience.

Prerequisites

  • Metamask extension installed and configured
  • Custom network added to the Metamask extension
  • Native assets defined for the custom network
