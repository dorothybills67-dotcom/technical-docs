Metamask Extension Bug Assets Native Assets For Custom Networks

Symptom

  • Users are unable to view native assets for custom networks when selected in the Metamask extension, resulting in incomplete asset displays.

Technical Root Cause

  • The issue arises from a bug in the Metamask extension's asset handling mechanism for custom networks, where it fails to fetch or display the native assets correctly.

Manual Overrides

  • To troubleshoot, try the following steps:
  • Check the custom network configuration to ensure it includes the native asset details.
  • Manually add the native asset using the Metamask API, utilizing a command similar to the one below: bash curl -X POST \ https://example.metamask.io/api/v1/assets \ -H 'Content-Type: application/json' \ -d '{"asset": "Native Asset", "network": "Custom Network"}'
  • Verify that the Metamask extension is updated to the latest version, as updates often include bug fixes for such issues. For more complex scenarios, consider utilizing an external console for deeper inspection and debugging.

Prerequisites

  • Metamask extension installed and configured
  • Custom network added to the Metamask extension
  • Basic understanding of Metamask API and asset management
