Metamask Extension Bug Assets Native Assets For Custom Networks

Symptom

  • Users are unable to view native assets for custom networks when selected in the Metamask extension, resulting in an incomplete asset list.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's asset handling mechanism for custom networks, where it fails to fetch and display the native assets.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Update the Metamask extension to the latest version
  • Manually add the native asset token using the token.address and token.symbol in the following JSON format:
{
  "address": "0x...token address",
  "symbol": "...token symbol",
  "decimals": 18,
  "image": "...token image URL"
}
  • Reset the Metamask extension's asset cache by deleting the assetCache.json file Consider using an external console to execute the override commands for more flexibility.

Prerequisites

  • Metamask extension installed and configured
  • Custom network added to the Metamask extension
  • Native asset token details (address, symbol, decimals, image) available
