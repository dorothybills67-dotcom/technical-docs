Metamask Extension Bug Assets Native Assets For Custom Networks

Symptom

  • Users are unable to view native assets for custom networks when selected, resulting in an incomplete asset list within the Metamask extension.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's handling of custom network configurations, specifically with regards to native asset detection and display.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Update the Metamask extension to the latest version
  • Manually add the native asset for the custom network using the asset's contract address and decimal places
  • Utilize the following JSON configuration to force-enable native asset detection:
{
  "network": "custom",
  "nativeAsset": {
    "symbol": "ETH",
    "decimals": 18
  }
}

Consider using an external console to further diagnose and resolve the issue.

Prerequisites

  • Metamask extension installed and configured
  • Custom network added to the Metamask extension
  • Native asset contract address and decimal places known
