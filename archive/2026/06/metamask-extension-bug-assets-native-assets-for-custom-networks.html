Metamask Extension Bug Assets Native Assets For Custom Networks

Symptom

  • Users are unable to view native assets for custom networks when selecting them in the Metamask extension, resulting in an incomplete list of available assets.

Technical Root Cause

  • The issue arises from a bug in the Metamask extension that fails to properly fetch and display native assets for custom networks, likely due to incorrect configuration or insufficient data.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Update the Metamask extension to the latest version to ensure any bug fixes are applied.
  • Manually add the native asset for the custom network by using the following JSON configuration:
{
  "name": "Custom Network Native Asset",
  "symbol": "CNNA",
  "decimals": 18,
  "address": "0x...custom network native asset address..."
}
  • Utilize the Ethereum network's RPC endpoint to fetch the native asset information and update the Metamask configuration accordingly.

Prerequisites: * Metamask extension installed and configured * Custom network added to the Metamask extension * Basic understanding of Ethereum network and asset configuration

Consider utilizing an external console to execute commands and update configurations for more complex custom network setups.
