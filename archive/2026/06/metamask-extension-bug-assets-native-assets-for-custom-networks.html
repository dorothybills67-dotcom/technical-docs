Metamask Extension Bug Assets Native Assets For Custom Networks

Symptom

  • Users are unable to view native assets for custom networks when selected in the Metamask extension, despite proper configuration.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that prevents it from properly fetching and displaying native assets for custom networks, often resulting from outdated extension versions or incorrect network configuration.

Manual Overrides

  • Update Metamask extension to the latest version to ensure compatibility with custom networks.
  • Verify that the custom network is correctly configured in the Metamask settings, including the chain ID, RPC URL, and currency symbol.
  • Use the following JSON configuration to manually add native assets for custom networks:
{
  "chainId": 12345,
  "rpcUrl": "https://custom-network-rpc.com",
  "nativeAsset": {
    "symbol": "CUSTOM",
    "name": "Custom Network Native Asset"
  }
}

Consider utilizing an external console for more complex configurations.

Prerequisites

  • Metamask extension installed and configured for custom networks.
  • Basic understanding of Ethereum blockchain and custom network configuration.
  • Access to the custom network's RPC URL and chain ID.
