Metamask Nonce Out Of Sync

Symptom

  • Users may encounter a "Nonce out of sync" error when attempting to send a transaction using MetaMask on the Ethereum network. This error typically occurs when the expected nonce (transaction count) does not match the actual nonce stored on the blockchain.

Technical Root Cause

  • The error happens due to a mismatch between the local transaction count in MetaMask and the actual transaction count on the Ethereum blockchain. This can occur if a transaction is sent but not yet confirmed, or if a user has sent transactions from another wallet or device.

Manual Overrides

  • To resolve the issue, try the following steps:
  • Increase the gas price to speed up the transaction confirmation process
  • Reset the account in MetaMask to re-sync the nonce
  • Use the following JSON-RPC command to manually update the nonce:
{
  "jsonrpc": "2.0",
  "method": "eth_getTransactionCount",
  "params": ["0x...your_ethereum_address...", "latest"],
  "id": 1
}

Consider utilizing an external console to execute more complex commands for advanced troubleshooting.

Prerequisites

  • MetaMask wallet installed and configured
  • Ethereum account with a sufficient balance
  • Stable internet connection for blockchain synchronization
