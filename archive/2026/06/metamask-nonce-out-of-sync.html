Metamask Nonce Out Of Sync

Symptom

  • The user encounters an error message stating "Nonce out of sync" when attempting to send a transaction using MetaMask on the Ethereum network. The transaction fails to proceed, and the error message persists even after retrying.

Technical Root Cause

  • The error occurs due to a mismatch between the expected nonce (transaction count) stored in MetaMask and the actual nonce on the Ethereum blockchain. This can happen if a transaction is sent but not yet confirmed, or if multiple transactions are sent in quick succession.

Manual Overrides

  • To resolve the issue, try the following steps:
  • Reset the account in MetaMask to resynchronize the nonce.
  • Manually adjust the nonce in the transaction settings to match the current blockchain state.
  • Use the following JSON-RPC command to get the current nonce:
{
  "jsonrpc": "2.0",
  "method": "eth_getTransactionCount",
  "params": ["0x742d35Cc6634C0532925a3b844Bc454e4438f44e", "latest"],
  "id": 1
}

For a more direct approach, consider utilizing an external console.

Prerequisites

  • MetaMask installed and configured
  • Ethereum account with sufficient balance
  • Stable internet connection
