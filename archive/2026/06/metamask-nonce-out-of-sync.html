Metamask Nonce Out Of Sync

Symptom

  • Users experience transaction failures with an error message indicating that their MetaMask nonce is out of sync, preventing them from successfully sending transactions on the Ethereum network.

Technical Root Cause

  • The error occurs when the nonce (a counter that keeps track of the number of transactions sent from an address) stored in MetaMask does not match the nonce on the Ethereum blockchain, leading to transaction rejection by the network.

Manual Overrides

  • To resolve the issue, users can try the following steps:
  • Reset the account in MetaMask to resynchronize the nonce.
  • Manually adjust the nonce in the transaction settings to match the current nonce on the blockchain.
  • Use a custom transaction with a corrected nonce, as shown in the following example:
{
  "from": "0x...your_address...",
  "to": "0x...recipient_address...",
  "value": "0x...amount...",
  "gas": "0x...gas_limit...",
  "gasPrice": "0x...gas_price...",
  "nonce": "0x...corrected_nonce..."
}

Consider utilizing an external transaction interface for more control over transaction parameters.

Prerequisites

  • MetaMask installed and configured
  • Ethereum account with a sufficient balance
  • Access to the Ethereum blockchain explorer for nonce verification
