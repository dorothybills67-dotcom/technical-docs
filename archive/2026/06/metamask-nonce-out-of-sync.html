Symptom

  • The user encounters a "Nonce out of sync" error when attempting to send a transaction using MetaMask on the Ethereum (ETH) network, indicating a discrepancy between the expected and actual nonce values.

Technical Root Cause

  • This error occurs due to a mismatch between the nonce stored in MetaMask and the nonce recorded on the Ethereum blockchain, often resulting from cancelled or pending transactions that have not been properly cleared.

Manual Overrides

  • Reset the account in MetaMask to resynchronize the nonce:
  • Attempt to send a new transaction with a high gas price to clear any pending transactions
  • Utilize the MetaMask settings to manually adjust the nonce
  • Employ the following command in a JSON-RPC console or equivalent:
{"jsonrpc":"2.0","method":"eth_sendTransaction","params":[{"from":"0x...","to":"0x...","value":"0x0","gas":"0x...","gasPrice":"0x...","nonce":"0x..."}]}

For more complex issues, consider accessing the transaction directly via an external interface for deeper inspection.

Prerequisites

  • Access to the Ethereum network
  • A MetaMask wallet installed and configured
  • Basic understanding of JSON-RPC commands and Ethereum transaction structure
