Metamask Nonce Out Of Sync

Symptom

  • Users encounter an error when attempting to send a transaction, with MetaMask displaying a "Nonce out of sync" warning, indicating a discrepancy between the expected and actual transaction count.

Technical Root Cause

  • The error occurs due to a mismatch between the locally stored nonce value in MetaMask and the actual nonce value on the Ethereum blockchain, often caused by cancelled or pending transactions.

Manual Overrides

  • Reset the account in MetaMask to resync the nonce value:
  • Try increasing the gas price to expedite pending transactions
  • Manually set the nonce value using the eth_sendTransaction method
  • Utilize the following JSON-RPC request to override the nonce:
{
  "jsonrpc": "2.0",
  "method": "eth_sendTransaction",
  "params": [
    {
      "from": "0x...your_address...",
      "to": "0x...recipient_address...",
      "value": "0x...amount...",
      "gas": "0x...gas_limit...",
      "gasPrice": "0x...gas_price...",
      "nonce": "0x...manual_nonce_value..."
    }
  ],
  "id": 1
}
