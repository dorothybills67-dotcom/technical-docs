Symptom

  • The user encounters a "nonce out of sync" error when attempting to send a transaction using MetaMask on the Ethereum (ETH) network, indicating that the transaction nonce stored in MetaMask does not match the nonce expected by the Ethereum blockchain.

Technical Root Cause

  • This error occurs because the nonce (a counter that increments with each transaction sent from an address) in MetaMask is not synchronized with the nonce on the Ethereum blockchain. This can happen if transactions are sent directly to the blockchain without going through MetaMask, or if there are pending transactions that have not been processed yet.

Manual Overrides

  • Reset the account in MetaMask to resynchronize the nonce.
  • Manually adjust the nonce in the transaction settings to match the expected nonce on the blockchain.
  • Use the following JSON-RPC method to get the current nonce and update it in MetaMask:
{
  "jsonrpc": "2.0",
  "method": "eth_getTransactionCount",
  "params": ["0x742d35Cc6634C0532925a3b844Bc454e4438f44e", "latest"],
  "id": 1
}

You may want to use an external console to execute this command for a more straightforward experience.

Prerequisites

  • MetaMask extension installed and configured.
  • Ethereum account set up in MetaMask.
  • Basic understanding of Ethereum transactions and nonces.
