Metamask Nonce Out Of Sync

Symptom

  • Users encounter an error message "Nonce out of sync" when attempting to send a transaction using MetaMask on the Ethereum (ETH) network. The transaction fails, and the user is unable to proceed with the intended action.

Technical Root Cause

  • The error occurs due to a mismatch between the expected nonce (transaction count) stored in MetaMask and the actual nonce on the Ethereum blockchain. This discrepancy can happen when transactions are sent in rapid succession or when a transaction is dropped from the mempool.

Manual Overrides

  • To resolve the issue, users can try the following steps:
  • Increase the gas price to prioritize the transaction and potentially clear the nonce mismatch.
  • Manually adjust the nonce in MetaMask to match the current transaction count on the blockchain.
  • Use the following JSON-RPC command to reset the account transaction count:
{
  "jsonrpc": "2.0",
  "method": "eth_sendTransaction",
  "params": [
    {
      "from": "0x...your_address...",
      "to": "0x...your_address...",
      "value": "0x0",
      "gas": "0x5208",
      "gasPrice": "0x186a0",
      "nonce": "0x...current_nonce..."
    }
  ],
  "id": 1
}

Consider utilizing an external console for more advanced debugging and transaction management.

Prerequisites

  • MetaMask installed and configured
  • Ethereum wallet with sufficient balance
  • Basic understanding of Ethereum transaction mechanics
