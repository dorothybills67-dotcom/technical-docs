Symptom

  • The user encounters an error message stating "MetaMask nonce out of sync" when attempting to send a transaction on the Ethereum network, indicating a discrepancy between the expected and actual nonce values.

Technical Root Cause

  • The error occurs due to a mismatch between the nonce value stored in MetaMask and the actual nonce value on the Ethereum blockchain, often resulting from concurrent transactions or network delays.

Manual Overrides

  • Reset the MetaMask account to resync the nonce:
  • Delete the MetaMask local storage
  • Reset the account on the MetaMask settings page
  • Use the following command in the console to directly reset the nonce:
eth_sendTransaction({from: '0x...your_address...', to: '0x...your_address...', value: 0, gas: 20000, gasPrice: 20000000000, nonce: '0x...expected_nonce...'})

Consider using an external console for more advanced debugging and transaction manipulation.

Prerequisites

  • A basic understanding of Ethereum transactions and nonce values
  • Access to the MetaMask wallet and its settings
  • Familiarity with the Ethereum blockchain explorer for verifying transaction history
