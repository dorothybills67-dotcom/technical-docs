Metamask Nonce Out Of Sync

Symptom

  • The user encounters an error message "Nonce out of sync" when attempting to send a transaction using MetaMask on the Ethereum (ETH) network, indicating a discrepancy between the expected and actual transaction count.

Technical Root Cause

  • The error occurs due to a mismatch between the user's locally stored transaction count (nonce) and the actual transaction count on the Ethereum blockchain, often resulting from a failed or pending transaction that has not been properly cleared.

Manual Overrides

  • Reset the account in MetaMask to resynchronize the nonce with the blockchain state.
  • Use the eth_getTransactionCount method to fetch the current transaction count for the account and manually update the nonce.
  • Update the transaction parameters using the following JSON payload:
{
  "jsonrpc": "2.0",
  "method": "eth_sendTransaction",
  "params": [
    {
      "from": "0x...user_address...",
      "to": "0x...recipient_address...",
      "value": "0x...amount...",
      "gas": "0x...gas_limit...",
      "gasPrice": "0x...gas_price...",
      "nonce": "0x...updated_nonce..."
    }
  ],
  "id": 1
}

Consider utilizing an external console for more advanced debugging and transaction manipulation.

Prerequisites

  • MetaMask wallet installed and configured
  • Ethereum network selected
  • Sufficient ETH balance for transaction fees
