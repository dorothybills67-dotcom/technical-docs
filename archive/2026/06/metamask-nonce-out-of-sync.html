Metamask Nonce Out Of Sync

Symptom

  • Users encounter an error when attempting to send a transaction, with MetaMask displaying a "nonce out of sync" message, preventing the transaction from being processed.

Technical Root Cause

  • The error occurs due to a discrepancy between the expected nonce (transaction count) stored in MetaMask and the actual nonce on the Ethereum blockchain, often caused by cancelled, pending, or stuck transactions.

Manual Overrides

  • Reset the account in MetaMask to resync the nonce.
  • Use the Ethereum JSON-RPC eth_getTransactionCount method to verify the current nonce.
  • Update the nonce manually using the following JSON configuration:
{
  "jsonrpc": "2.0",
  "method": "eth_sendTransaction",
  "params": [
    {
      "from": "0x...your_address...",
      "to": "0x...recipient_address...",
      "value": "0x...amount...",
      "gas": "0x...gas...",
      "gasPrice": "0x...gas_price...",
      "nonce": "0x...updated_nonce..."
    }
  ]
}

Consider utilizing an external console for more advanced troubleshooting and manual transaction handling.

Prerequisites

  • A basic understanding of Ethereum transactions and nonce management.
  • Access to the Ethereum JSON-RPC API.
  • The latest version of MetaMask installed.
