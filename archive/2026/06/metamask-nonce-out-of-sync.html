Symptom

  • The user encounters a "nonce out of sync" error when attempting to send a transaction using MetaMask on the Ethereum (ETH) network. This error typically occurs when the nonce value stored in MetaMask does not match the nonce value expected by the Ethereum network.

Technical Root Cause

  • The error happens due to a discrepancy between the transaction count in MetaMask and the actual transaction count on the Ethereum blockchain. This can occur if a transaction is sent but not confirmed, or if a transaction is cancelled or replaced.

Manual Overrides

  • To resolve the issue, try the following steps:
  • Reset the account in MetaMask to resynchronize the nonce value.
  • Manually set the nonce value to the correct number using the Ethereum JSON-RPC API.
  • Use the following JSON-RPC command to get the current transaction count:
{
  "jsonrpc": "2.0",
  "method": "eth_getTransactionCount",
  "params": ["0x742d35Cc6634C0532925a3b844Bc454e4438f44e", "latest"],
  "id": 1
}

Consider utilizing an external console for more advanced troubleshooting and direct access to Ethereum network data.
