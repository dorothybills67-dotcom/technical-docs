TRON Post-Quantum Signature Migration Error

Symptom

  • Users may encounter issues with transaction verification and validation on the TRON network after attempting to migrate to post-quantum signatures, resulting in failed transactions and error messages indicating signature verification failures.

Technical Root Cause

  • The error occurs due to incompatible signature schemes between the user's wallet and the TRON network, which has transitioned to post-quantum cryptography for enhanced security.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Update the wallet software to the latest version compatible with post-quantum signatures.
  • Configure the wallet to use the new post-quantum signature scheme:
{
  "signatureScheme": "postQuantum",
  "cryptographyVersion": "v2"
}
  • Perform a full node resync to ensure the local node is updated with the latest blockchain data.

Prerequisites: * Ensure the TRON network is fully synchronized. * Verify the wallet software is compatible with post-quantum signatures. * Backup wallet data before attempting any manual overrides. For further troubleshooting, consider utilizing an external console to inspect node and wallet configurations.
