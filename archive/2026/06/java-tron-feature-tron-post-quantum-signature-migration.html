TRON Post-Quantum Signature Migration Failure

Symptom

  • Users may experience errors when attempting to migrate their TRON signatures to post-quantum signatures, resulting in failed transactions and loss of access to their accounts.

Technical Root Cause

  • The error occurs due to incompatible signature formats between the legacy TRON network and the new post-quantum signature scheme, causing the java-tron client to malfunction.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Update the java-tron client to the latest version
  • Configure the post-quantum signature scheme manually using the following JSON configuration:
{
  "signatureScheme": "post-quantum",
  "migrationMode": "enabled"
}
  • Run the migration tool with the --force flag to override any compatibility issues For a smooth migration, consider utilizing an external console for more advanced troubleshooting.

Prerequisites

  • Java-tron client version 4.5 or higher
  • TRON network updated to support post-quantum signatures
  • Compatible wallet software for post-quantum signature management
