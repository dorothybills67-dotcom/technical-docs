TRON Post-Quantum Signature Migration Failure

Symptom

  • Users encounter an error when attempting to migrate to post-quantum signatures on the TRON network using java-tron, resulting in failed transactions and inability to utilize the new signature scheme.

Technical Root Cause

  • The error occurs due to incomplete or incorrect configuration of the post-quantum signature parameters in the java-tron implementation, leading to a mismatch between the expected and actual cryptographic algorithms used.

Manual Overrides

  • To resolve the issue, apply the following manual overrides:
  • Update the tron.properties file to include the correct post-quantum signature configuration.
  • Use the --post-quantum flag when running the java-tron node to enable the new signature scheme.
  • Verify that the TRON network is configured to support post-quantum signatures by checking the network's configuration using the following code block:
{
  "network": {
    "postQuantum": true,
    "signatureScheme": "SPHINCS"
  }
}

Prerequisites: * Java-tron node version 4.5.0 or higher * TRON network configured to support post-quantum signatures * Correct post-quantum signature configuration in tron.properties file For further troubleshooting, consider using an external console to inspect the node's configuration and logs.
