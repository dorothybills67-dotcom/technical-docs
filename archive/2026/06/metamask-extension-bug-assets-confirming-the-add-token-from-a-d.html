Metamask Extension Bug Assets Confirming The Add Token From A Dapp Dialog

Symptom

  • Users encounter a persistent "Add token" page when attempting to confirm the addition of a token from a decentralized application (dapp) using the Metamask extension, with the dialog box failing to proceed beyond the confirmation step.

Technical Root Cause

  • The issue arises from a known bug within the Metamask extension, specifically related to the handling of token additions from dapps, which causes the confirmation dialog to loop indefinitely.

Manual Overrides

  • Attempt to manually add the token by inputting its contract address and other necessary details directly into Metamask.
  • Clear the browser's cache and cookies, then restart the browser to ensure a fresh environment.
  • Utilize the following JSON configuration to reset Metamask settings:
{
  "provider": {
    "chainId": 1
  },
  "tokens": []
}

Consider utilizing an external console for deeper inspection and potential manual intervention.

Prerequisites

  • Ensure Metamask is updated to the latest version.
  • Verify that the dapp and its associated smart contracts are functioning correctly.
  • Confirm the token's contract address and details are accurate and correctly formatted.
