Symptom

  • Users encounter an issue when using the Metamask extension to sign a message, specifically with the Bitcoin Snap Sign Message configuration. Upon clicking the back button during this process, the Metamask extension becomes unresponsive or throws an error.

Technical Root Cause

  • The error occurs due to a timing issue between the Metamask extension's internal state management and the external Bitcoin Snap Sign Message module. When the back button is clicked, the extension's state is not properly reset, leading to a conflict with the ongoing signing process.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Clear the Metamask extension's cache and storage
  • Restart the browser and reload the Metamask extension
  • Execute the following command to reset the extension's internal state:
mm-reset --sign-message-config

Consider utilizing an external console for further debugging and customization.

Prerequisites

  • Metamask extension version 10.11.0 or later
  • Bitcoin Snap Sign Message module enabled
  • Ethereum network (ETH) selected in the Metamask extension
