Metamask Extension Bug Sign Message Clicking Back Button In The

Symptom

  • When using the Metamask extension to sign a message, clicking the back button in the Bitcoin Snap Sign Message confirmation window causes the extension to become unresponsive, and the signing process is interrupted.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that fails to handle the back button click event properly, leading to a loop where the signing process is not terminated correctly.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Clear the browser cache and restart the browser
  • Disable and re-enable the Metamask extension
  • Use the following JSON configuration to override the default signing behavior:
{
  "signing": {
    "handleBackButton": true
  }
}

Consider utilizing an external console for advanced debugging and configuration.

Prerequisites

  • Bitcoin Snap Sign Message feature enabled
  • Latest browser version updated
