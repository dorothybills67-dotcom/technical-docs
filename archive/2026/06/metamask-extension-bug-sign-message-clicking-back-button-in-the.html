Symptom

  • Users experience a bug when using the Metamask extension to sign a message, specifically when clicking the back button in the Bitcoin Snap Sign Message confirmation dialog. The error prevents the user from successfully signing the message, resulting in a frustrating experience.

Technical Root Cause

  • The bug is caused by a conflict between the Metamask extension's handling of the sign message flow and the Bitcoin Snap's confirmation dialog. When the user clicks the back button, the Metamask extension fails to properly handle the cancellation of the sign message request, leading to the error.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Check the Metamask extension version and update to the latest version if necessary
  • Clear the browser cache and restart the browser
  • Use the following command to reset the Metamask extension's sign message flow:
mm-reset-sign-message-flow --ethereum-network mainnet

Consider utilizing an external console for a more seamless troubleshooting experience.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum network (mainnet) selected
  • Bitcoin Snap Sign Message confirmation dialog enabled
