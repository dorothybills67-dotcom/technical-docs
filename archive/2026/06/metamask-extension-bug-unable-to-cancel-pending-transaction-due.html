Metamask Extension Bug Unable To Cancel Pending Transaction Due

Symptom

  • Users are unable to cancel pending transactions in Metamask, resulting in an error message stating that there is not enough gas, despite having sufficient balance.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, which incorrectly calculates the gas required to cancel a pending transaction, leading to a gas shortage error.

Manual Overrides

  • Increase the gas limit manually by using the following commands:
  • eth_estimateGas to estimate the required gas for the cancellation transaction
  • eth_sendTransaction with the increased gas limit to override the automatic calculation
  • eth_cancelPendingTransaction to cancel the pending transaction with the new gas settings
curl -X POST \
  http://localhost:8545 \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","method":"eth_sendTransaction","params":[{"from":"0x...","to":"0x...","gas":"0x...","gasPrice":"0x..."}],"id":1}'

Prerequisites: * Metamask extension installed and configured * Sufficient ETH balance to cover the transaction fees * A basic understanding of Ethereum transaction mechanics For further troubleshooting, consider utilizing an external Ethereum console to execute the above commands.
