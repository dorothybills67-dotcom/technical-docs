Metamask Extension Bug Unable To Cancel Pending Transaction Due

Symptom

  • Users are unable to cancel pending transactions in Metamask, resulting in a persistent error message indicating that there is not enough gas to complete the cancellation process.

Technical Root Cause

  • The error occurs when the pending transaction's gas price is higher than the current network gas price, causing Metamask to fail in canceling the transaction due to insufficient gas.

Manual Overrides

  • Increase the gas limit for the cancellation transaction:
  • Use a higher gas price for the cancellation transaction
  • Utilize the eth_cancel method with a custom gas price
  • Execute the following command to override the gas price:
eth_sendTransaction({from: '0x...your_address...', to: '0x...contract_address...', gas: '0x...higher_gas...', gasPrice: '0x...custom_gas_price...'})

Consider using an external Ethereum console for more advanced troubleshooting and control.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum node or provider (e.g. Infura, Alchemy) connection
  • Basic understanding of Ethereum transaction mechanics
