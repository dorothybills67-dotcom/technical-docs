Metamask Extension Bug Unable To Cancel Pending Transaction Due

Symptom

  • Users are unable to cancel pending transactions using the Metamask extension, resulting in an error message indicating that there is not enough gas.

Technical Root Cause

  • The error occurs because the pending transaction is waiting for confirmation on the Ethereum blockchain, and Metamask is unable to override the transaction due to insufficient gas.

Manual Overrides

  • Increase the gas limit in the Metamask settings to allow for overriding of the pending transaction.
  • Use the Ethereum wallet's transaction replacement feature by increasing the gas price.
  • Utilize the following JSON-RPC method to replace the pending transaction:
{
  "jsonrpc": "2.0",
  "method": "eth_sendTransaction",
  "params": [
    {
      "from": "0x...your-ethereum-address...",
      "to": "0x...recipient-address...",
      "value": "0x...value-in-wei...",
      "gas": "0x...gas-limit...",
      "gasPrice": "0x...gas-price-in-wei..."
    }
  ],
  "id": 1
}

Consider using an external console to execute the JSON-RPC method for more flexibility.

Prerequisites

  • Metamask extension installed and configured
  • Sufficient Ethereum balance to cover gas costs
  • Basic understanding of Ethereum transaction management
