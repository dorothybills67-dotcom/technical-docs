Metamask Extension Bug Batch Sell No Warning Appears When Batch

Symptom

  • Users do not receive a warning when attempting to batch sell a single token on the Review page, which can lead to unintended transactions.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, where the warning prompt is not triggered when batch selling a single token, potentially resulting in unexpected behavior.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Check the transaction details carefully before confirming the batch sell.
  • Use the Metamask console to verify the transaction parameters.
  • Run the following command to override the batch sell warning:
eth_sendTransaction({from: '0x...user_address...', to: '0x...contract_address...', value: '0x...amount...'})

Consider utilizing an external console for more advanced troubleshooting and transaction management.
