Metamask Extension Using Waituntil To Ensure All Scripts Load

Symptom

  • Users may experience delayed or failed loading of web pages when using the Metamask extension, resulting in incomplete functionality or errors.

Technical Root Cause

  • The issue arises from the Metamask extension's use of the waitUntil function to ensure all scripts are loaded before allowing the page to proceed, which can cause timing conflicts and delays.

Manual Overrides

  • To resolve the issue, users can try the following overrides:
  • Increase the timeout value in the waitUntil function to allow more time for scripts to load.
  • Use a different loading strategy, such as loading scripts asynchronously.
  • Implement a retry mechanism to handle failed script loads.
mm_wait_until_timeout=30000
