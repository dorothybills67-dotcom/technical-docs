Symptom

  • The user encounters a "Critical TRC-20 energy spike error" message, indicating an abnormal increase in energy consumption by a TRC-20 smart contract on the TRON network.

Technical Root Cause

  • This error occurs when a smart contract's energy consumption exceeds the maximum allowed limit, causing the contract's execution to fail and resulting in the error message.

Manual Overrides

  • Adjust the contract's energy limit by modifying the energy_limit parameter in the contract's settings.
  • Optimize the contract's code to reduce energy consumption by minimizing unnecessary computations and data storage.
  • Update the contract's configuration using the following JSON code block:
{
  "contract_address": "TX......",
  "energy_limit": 10000,
  "parameters": {
    "optimizer_enabled": true
  }
}

Consider utilizing an external console for more granular control over contract configurations.

Prerequisites

  • TRON wallet with sufficient funds
  • TRON network node with the latest software version
  • Basic understanding of smart contract development and deployment on TRON
