Metamask Extension Bug Assets Manage Tokens Miss Match With Net

Symptom

  • Users experience a mismatch between the tokens displayed in the "Manage Tokens" section of the Metamask extension and the network filter, resulting in incorrect token listings.

Technical Root Cause

  • The error occurs due to a desynchronization between the Metamask extension's internal token registry and the Ethereum network's token listings, often caused by outdated extension versions, incorrect network configurations, or incomplete token data.

Manual Overrides

  • Update the Metamask extension to the latest version and restart the browser.
  • Reset the Metamask token registry by executing the following command:
mm-token-registry reset --network mainnet
  • Manually add the missing tokens using the token contract address and decimal value.

Prerequisites: * Metamask extension installed and configured for the Ethereum network * Basic understanding of Ethereum token contracts and decimal values * Latest version of the browser and Metamask extension

For advanced troubleshooting, consider utilizing an external console to execute low-level commands and query the Ethereum network directly.
