Symptom

  • Users experience a failure when attempting to swap and bridge assets on the Arbitrum network using the Metamask extension, resulting in a transaction error.

Technical Root Cause

  • The error occurs due to a compatibility issue between the Metamask extension and the Arbitrum network, specifically with the implementation of the bridge and swap functionality.

Manual Overrides

  • Update the Metamask extension to the latest version to ensure compatibility with the Arbitrum network.
  • Configure the Arbitrum network settings manually in the Metamask extension:
{
  "chainId": 42161,
  "chainName": "Arbitrum",
  "nativeCurrency": {
    "name": "ETH",
    "symbol": "ETH",
    "decimals": 18
  },
  "rpcUrls": ["https://arb1.arbitrum.io/rpc"],
  "blockExplorerUrls": ["https://arbiscan.io"]
}

Prerequisites: * Metamask extension installed and configured * Arbitrum network added to the Metamask extension * ETH wallet balance sufficient for transaction fees
