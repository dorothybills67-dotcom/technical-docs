Metamask Extension Bug Swap And Bridge Fail On Arbitrum Network

Symptom

  • Users are experiencing failed swap and bridge transactions on the Arbitrum network when using the Metamask extension, resulting in an error message indicating the transaction cannot be completed.

Technical Root Cause

  • The error occurs due to a compatibility issue between the Metamask extension and the Arbitrum network, specifically with the handling of layer 2 scaling solutions.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Update the Metamask extension to the latest version
  • Configure the Arbitrum network settings manually in the Metamask extension
  • Execute the following json-rpc command to override the default gas price:
{
  "jsonrpc": "2.0",
  "method": "eth_sendTransaction",
  "params": [
    {
      "from": "0x...your-eth-address...",
      "to": "0x...contract-address...",
      "value": "0x...hex-value...",
      "gas": "0x...hex-gas...",
      "gasPrice": "0x...hex-gas-price..."
    }
  ],
  "id": 1
}

Consider utilizing an external console for more advanced debugging and transaction manipulation.

Prerequisites

  • Metamask extension installed and configured
  • Arbitrum network added to the Metamask extension
