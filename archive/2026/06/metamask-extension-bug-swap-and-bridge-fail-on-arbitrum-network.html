Metamask Extension Bug Swap And Bridge Fail On Arbitrum Network

Symptom

  • Users experience failed transactions when attempting to swap or bridge assets on the Arbitrum network using the Metamask extension, resulting in an error message indicating the swap and bridge functionality is not working as expected.

Technical Root Cause

  • The issue arises from a compatibility problem between the Metamask extension and the Arbitrum network, likely due to outdated extension versions or incorrect network configuration.

Manual Overrides

  • Check for updates to the Metamask extension and install the latest version.
  • Configure the Arbitrum network manually in Metamask by adding the network details:
{
  "chainId": 42161,
  "chainName": "Arbitrum",
  "rpcUrls": ["https://arb1.arbitrum.io/rpc"],
  "nativeCurrency": {
    "name": "Ether",
    "symbol": "ETH",
    "decimals": 18
  }
}
  • Reset the Metamask extension to its default settings to remove any potentially conflicting configurations.

Prerequisites: - Metamask extension installed - Arbitrum network added to Metamask - Sufficient ETH for transaction fees
