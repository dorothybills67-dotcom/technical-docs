Metamask Extension Bug Swap And Bridge Fail On Arbitrum Network

Symptom

  • Users are experiencing failures when attempting to swap and bridge assets on the Arbitrum network using the Metamask extension, resulting in failed transactions and potential loss of funds.

Technical Root Cause

  • The error occurs due to a compatibility issue between the Metamask extension and the Arbitrum network, specifically with the latest protocol updates and contract deployments.

Manual Overrides

  • Update the Metamask extension to the latest version and restart the browser
  • Manually configure the Arbitrum network settings in Metamask to use the correct RPC endpoint and chain ID
  • Use the following JSON configuration to override the default settings:
{
  "chainId": 42161,
  "chainName": "Arbitrum",
  "rpcUrl": "https://arb1.arbitrum.io/rpc"
}

Consider utilizing an external console for further debugging and configuration.

Prerequisites

  • Metamask extension installed and configured
  • Arbitrum network added to the Metamask extension
  • Latest version of the browser and Metamask extension
