Metamask Extension Bug Swap And Bridge Fail On Arbitrum Network

Symptom

  • Users experience a failure when attempting to swap or bridge assets on the Arbitrum network using the Metamask extension, resulting in an error message indicating the operation cannot be completed.

Technical Root Cause

  • The error occurs due to a compatibility issue between the Metamask extension and the Arbitrum network, potentially caused by outdated extension versions, incorrect network configuration, or insufficient gas limits.

Manual Overrides

  • Update Metamask extension to the latest version and restart the browser.
  • Configure the Arbitrum network manually in Metamask using the following JSON configuration:
{
  "chainId": 42161,
  "chainName": "Arbitrum",
  "nativeCurrency": {
    "name": "ETH",
    "symbol": "ETH",
    "decimals": 18
  },
  "rpcUrls": ["https://arb1.arbitrum.io/rpc"],
  "blockExplorerUrls": ["https://arbiscan.io"]
}
  • Increase the gas limit for transactions on the Arbitrum network to ensure sufficient resources for the swap and bridge operations. Consider utilizing an external console for advanced debugging and configuration.

Prerequisites

  • Metamask extension installed and configured.
  • Arbitrum network added to Metamask.
  • Sufficient ETH balance for gas fees.
