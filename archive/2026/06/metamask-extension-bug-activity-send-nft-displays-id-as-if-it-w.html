Metamask Extension Bug Activity Send Nft Displays Id As If It Were Amount

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem notice that when attempting to send an NFT, the activity log displays the NFT's ID as if it were the amount being sent, rather than showing the actual NFT being transferred.

Technical Root Cause

  • This error occurs due to a bug in the Metamask extension's handling of NFT transfer activities, where it mistakenly treats the NFT's unique identifier as a quantity.

Manual Overrides

  • To temporarily resolve this issue, users can try the following:
  • Check for and install any available updates for the Metamask extension to ensure they are running the latest version.
  • Manually verify the transaction details on the Ethereum blockchain explorer to confirm the correct NFT is being sent.
  • Use the following JSON payload to override the display settings:
{
  "transaction": {
    "from": "0x... sender address ...",
    "to": "0x... recipient address ...",
    "value": "0",
    "data": "... NFT transfer data ..."
  }
}

For more complex transactions, consider using an external tool for crafting and verifying the transaction details.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum wallet with sufficient funds for transaction fees
  • NFT to be sent is properly configured and owned by the sender
