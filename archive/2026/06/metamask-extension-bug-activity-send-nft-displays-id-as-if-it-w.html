Metamask Extension Bug Activity Send Nft Displays Id As If It Were Amount

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem encounter a bug where the activity log for sending NFTs displays the NFT's ID as if it were the amount being sent, instead of showing the actual amount of cryptocurrency or tokens transferred.

Technical Root Cause

  • This error occurs due to a misinterpretation of data types within the Metamask extension's code, where the NFT's unique identifier is mistakenly treated as a quantity.

Manual Overrides

  • To temporarily resolve this issue, users can try the following steps:
  • Check for updates to the Metamask extension to ensure they are running the latest version.
  • Clear the browser's cache and cookies to remove any potentially corrupted data.
  • Use the following command to reset the Metamask extension's settings:
mm-reset --hard

Consider using an external console for more complex troubleshooting steps.

Prerequisites

  • Metamask extension installed and configured on the browser.
  • An Ethereum wallet set up and connected to the Metamask extension.
  • The latest version of the browser being used.
