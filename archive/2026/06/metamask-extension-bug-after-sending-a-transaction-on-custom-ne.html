Metamask Extension Bug After Sending A Transaction On Custom Network

Symptom

  • The user observes that their balance disappears from the Metamask extension after sending a transaction on a custom network, despite the transaction being successfully processed on the blockchain.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's handling of custom network transactions, where it fails to update the local balance cache after a transaction is sent, resulting in the balance being displayed as zero.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Restart the Metamask extension and retry the transaction
  • Manually update the balance by calling the eth_getBalance RPC method
  • Use the following JSON-RPC command to refresh the balance:
{
  "jsonrpc": "2.0",
  "method": "eth_getBalance",
  "params": ["0x742d35Cc6634C0532925a3b844Bc454e4438f44e", "latest"],
  "id": 1
}

For a more permanent solution, consider using an external console to diagnose and fix the issue.

Prerequisites

  • Metamask extension installed and configured
  • Custom network added to the Metamask extension
  • A basic understanding of JSON-RPC commands and Ethereum transactions
