Metamask Extension Bug After Sending A Transaction On Custom Network

Symptom

  • The user observes that their balance disappears from the Metamask extension after sending a transaction on a custom network, despite the transaction being successfully processed on the blockchain.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that fails to properly update the balance after a transaction is sent on a custom network, leading to a discrepancy between the actual balance on the blockchain and the balance displayed in the extension.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Reset the Metamask account by removing and re-adding the custom network.
  • Update the Metamask extension to the latest version.
  • Use the following JSON-RPC command to fetch the latest balance:
{
  "jsonrpc": "2.0",
  "method": "eth_getBalance",
  "params": ["0x742d35Cc6634C0532925a3b844Bc454e4438f44e", "latest"],
  "id": 1
}

Consider utilizing an external console for more advanced debugging and troubleshooting.

Prerequisites

  • Metamask extension installed and configured
  • Custom network added to the Metamask extension
  • Basic understanding of JSON-RPC commands and blockchain transactions
