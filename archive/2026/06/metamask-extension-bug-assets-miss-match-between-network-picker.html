Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • The Metamask extension displays a mismatch between the selected network in the network picker and the assets shown in the modal, which is supposed to show "All popular" assets but instead shows assets from a different network.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that fails to update the asset list when the user switches between different Ethereum networks, resulting in a mismatch between the expected and actual assets displayed.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Refresh the Metamask extension and retry the asset selection process.
  • Manually update the asset list by adding or removing assets from the "All popular" list.
  • Execute the following JSON-RPC command to reset the asset cache:
{
  "jsonrpc": "2.0",
  "method": "eth_clearAssetCache",
  "params": [],
  "id": 1
}

For a more permanent fix, consider using an external console to diagnose and resolve the issue.

Prerequisites

  • Metamask extension installed and configured on the browser
  • Ethereum network selected and connected
  • Asset list populated with "All popular" assets
