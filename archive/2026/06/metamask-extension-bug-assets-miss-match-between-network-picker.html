Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • The Metamask extension displays a mismatch between the selected network in the network picker and the assets shown in the modal, with the modal showing "All popular" assets instead of the expected assets for the chosen network.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that causes the network picker and the asset modal to become desynchronized, resulting in the incorrect display of assets.

Manual Overrides

  • Try resetting the Metamask extension to its default settings using the mm-reset command.
  • Use the following JSON configuration to manually override the asset display:
{
  "network": "mainnet",
  "assets": ["ETH", "DAI", "USDC"]
}
  • Manually update the Metamask extension to the latest version using the mm-update command.

Prerequisites: * Metamask extension installed and enabled * Ethereum network selected in the network picker * Latest version of the Metamask extension available

For further debugging, consider using an external console to inspect the extension's internal state.
