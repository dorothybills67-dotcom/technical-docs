Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • The Metamask extension displays a mismatch between the selected network in the network picker and the assets shown in the modal, which incorrectly shows "All popular" assets instead of the assets specific to the chosen network.

Technical Root Cause

  • This error occurs due to a bug in the Metamask extension's logic for handling network changes and updating the asset list accordingly. The extension fails to properly sync the network selection with the asset display, leading to the mismatch.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Force-refresh the Metamask extension by restarting the browser or disabling and re-enabling the extension.
  • Clear the extension's local storage to reset its state:
{
  "network": "mainnet",
  "assets": []
}
  • Manually update the extension's configuration to ensure it is set to the correct network.

Prerequisites: * Metamask extension is installed and enabled * The user has a basic understanding of blockchain networks and assets * The user has access to the browser's developer tools
