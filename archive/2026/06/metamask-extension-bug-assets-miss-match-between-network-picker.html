Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users experience a mismatch between the assets displayed in the Metamask extension's network picker and the assets shown in the modal window, which is supposed to display All popular assets, but instead shows an incorrect list.

Technical Root Cause

  • The error occurs due to a desynchronization between the Metamask extension's internal state and the Ethereum network's actual asset listings, likely caused by a caching issue or an outdated asset registry.

Manual Overrides

  • Attempt to reset the Metamask extension's cache by running the following commands:
  • metamask.resetCache()
  • metamask.reloadAssets()
  • bash echo '{"jsonrpc":"2.0","method":"eth_accounts","params":[]}' | curl -X POST -H "Content-Type: application/json" https://mainnet.infura.io/v3/YOUR_PROJECT_ID Consider using an external console for more advanced debugging and troubleshooting.

Prerequisites

  • Metamask extension installed and enabled
  • Ethereum network connection established
  • Latest version of the Metamask extension installed
