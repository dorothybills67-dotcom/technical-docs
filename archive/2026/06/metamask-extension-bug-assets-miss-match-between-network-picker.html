Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • The Metamask extension displays a mismatch between the selected network in the network picker and the assets shown in the modal, which incorrectly shows "All popular" assets from a different network.

Technical Root Cause

  • The error occurs due to a temporary desynchronization between the Metamask extension's network configuration and the asset data fetched from the Ethereum blockchain, often caused by network congestion or outdated extension versions.

Manual Overrides

  • Try resetting the Metamask extension by reinstalling it and restarting the browser.
  • Use the eth_chainId RPC method to manually specify the correct chain ID for the desired network.
  • Update the extension's asset configuration using the following JSON data:
{
  "network": "mainnet",
  "chainId": 1,
  "assets": [
    {
      "address": "0x...",
      "symbol": "ETH",
      "decimals": 18
    }
  ]
}

For a more precise configuration, consider accessing the extension's internal state using an external console.

Prerequisites

  • Metamask extension version 10.23.1 or later
  • Ethereum network connection via Infura or a local node
  • Basic understanding of Ethereum blockchain and JSON data formats
