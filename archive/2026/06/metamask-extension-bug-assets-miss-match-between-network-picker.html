Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • Users experience a mismatch between the network selected in the Metamask extension's network picker and the assets displayed in the modal, which shows "All popular" assets regardless of the chosen network.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's logic for fetching and displaying assets based on the selected network, resulting in a discrepancy between the expected and actual assets shown.

Manual Overrides

  • Try resetting the Metamask extension to its default settings using the mm-reset command.
  • Manually configure the network and asset settings using the following JSON configuration:
{
  "networks": {
    "mainnet": {
      "chainId": 1,
      "assets": ["ETH", "DAI"]
    },
    "ropsten": {
      "chainId": 3,
      "assets": ["ETH", "USDT"]
    }
  }
}
  • Update the Metamask extension to the latest version using the mm-update command.

Prerequisites: * Metamask extension installed and enabled * Ethereum network selected * Latest version of the browser

For further debugging, consider using an external console to inspect the extension's internal state.
