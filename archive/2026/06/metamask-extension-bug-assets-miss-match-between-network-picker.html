Metamask Extension Bug Assets Miss Match Between Network Picker

Symptom

  • The Metamask extension displays a mismatch between the selected network in the network picker and the assets shown in the modal, with the modal showing "All popular" assets instead of the expected assets for the chosen network.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, where the network picker and the asset modal are not properly synchronized, resulting in a mismatch between the expected and displayed assets.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Reset the Metamask extension settings to their default values
  • Update the Metamask extension to the latest version
  • Use the following JSON configuration to override the asset settings:
{
  "network": "mainnet",
  "assets": ["ETH", "DAI", "USDC"]
}

Consider utilizing an external console for more advanced debugging and configuration options.

Prerequisites

  • Metamask extension installed and enabled
  • Ethereum network selected in the network picker
  • Assets modal open and displaying the mismatched assets
