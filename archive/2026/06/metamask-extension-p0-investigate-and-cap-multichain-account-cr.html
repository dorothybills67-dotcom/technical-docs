Metamask Extension P0 Investigate And Cap Multichain Account Creation/Syncing Span Fan-Out

Symptom

  • Users experience an error when creating or syncing multichain accounts in Metamask, resulting in a fan-out issue that affects the performance and stability of the extension.

Technical Root Cause

  • The error occurs due to an uncontrolled span of account creation and syncing across multiple chains, causing a resource overload and subsequent timeout.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Set a limit on the number of chains that can be synced simultaneously using the MAX_CHAINS environment variable.
  • Implement a rate limiter to control the frequency of account creation and syncing requests.
  • Use the following JSON configuration to cap the span fan-out:
{
  "multichain": {
    "maxChains": 5,
    "syncInterval": 30000
  }
}

Consider utilizing an alternative console to execute these commands for more flexibility.

  • Access to the Metamask extension's configuration files and environment variables
