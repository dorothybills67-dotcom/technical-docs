Metamask Extension Bug Sl Tp Percentages Change When Changing L

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem may notice that their Stop Loss (SL) and Take Profit (TP) percentages are altered when they change the leverage of their positions.

Technical Root Cause

  • The issue arises from a bug within the Metamask extension that incorrectly recalculates SL and TP percentages based on the new leverage, rather than maintaining the initially set values.

Manual Overrides

  • To temporarily resolve the issue, users can attempt the following manual overrides:
  • Edit the transaction parameters manually before confirming the transaction.
  • Use a third-party ETH wallet that does not exhibit this bug.
  • Utilize the following JSON configuration to override the default behavior:
{
  "slPercentage": 5,
  "tpPercentage": 10,
  "leverage": 2
}

Consider using an external console for more complex overrides and better control over transaction parameters.

Prerequisites

  • Basic understanding of Ethereum transactions and leverage
  • Familiarity with the Metamask extension and its functionality
  • Knowledge of JSON configuration files and their application in Ethereum transactions
