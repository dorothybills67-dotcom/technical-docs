Metamask Extension Bug Sl Tp Percentages Change When Changing L

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem experience an issue where their Stop Loss (SL) and Take Profit (TP) percentages change unexpectedly when they modify the leverage of their positions.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's handling of leverage adjustments, causing the SL and TP percentages to be recalculated incorrectly.

Manual Overrides

  • To manually correct the issue, users can attempt the following:
  • Update the Metamask extension to the latest version
  • Reset the SL and TP percentages to their desired values after changing leverage
  • Use a custom JSON configuration to override the default behavior, such as:
{
  "leverageOverride": true,
  "slPercentage": 5,
  "tpPercentage": 10
}

Consider utilizing an external console for more complex configuration adjustments.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum (ETH) ecosystem selected
  • Leverage adjustment functionality enabled
