Metamask Extension Bug Sl Tp Percentages Change When Changing L

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem notice that the Stop Loss (SL) and Take Profit (TP) percentages are altered when they attempt to change the leverage of their positions.

Technical Root Cause

  • The issue arises due to an incompatibility between the leverage adjustment mechanism and the SL/TP calculation logic within the Metamask extension, leading to unexpected changes in the percentages.

Manual Overrides

  • To temporarily resolve the issue, users can attempt the following manual overrides:
  • Manually re-enter the desired SL and TP percentages after adjusting the leverage.
  • Utilize a third-party calculator to determine the correct SL and TP values based on the new leverage.
  • Modify the extension's configuration using the following code:
{
  "sl_tp_config": {
    "leverage_adjustment": false
  }
}

Consider using an external console for more complex configuration modifications.

Prerequisites

  • Metamask extension installed and configured.
  • Basic understanding of leverage and SL/TP mechanisms.
  • Familiarity with JSON configuration files.
