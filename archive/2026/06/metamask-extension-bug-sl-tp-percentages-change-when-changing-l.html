Metamask Extension Bug Sl Tp Percentages Change When Changing L

Symptom

  • Users of the MetaMask extension on the Ethereum (ETH) ecosystem experience unexpected changes in Stop-Loss (SL) and Take-Profit (TP) percentages when adjusting leverage levels, despite no intentional modifications to these settings.

Technical Root Cause

  • The issue arises from a bug within the MetaMask extension that incorrectly recalculates SL and TP percentages based on the new leverage settings, rather than maintaining the originally set values.

Manual Overrides

  • To temporarily resolve this issue, users can attempt the following manual overrides:
  • Directly edit the transaction parameters in the extension's settings to reflect the desired SL and TP percentages.
  • Utilize a custom script to automate the adjustment of these parameters, ensuring consistency across different leverage levels.
  • Implement a workaround using the following code: json { "transaction": { "leverage": 5, "stopLoss": 10, "takeProfit": 15 } } Consider utilizing an external console for more complex adjustments.

Prerequisites

  • MetaMask extension installed and configured.
  • Basic understanding of leverage and SL/TP settings.
  • Familiarity with JSON formatting for custom script implementation.
