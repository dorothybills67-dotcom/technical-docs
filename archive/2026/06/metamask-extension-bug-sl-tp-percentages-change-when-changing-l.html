Metamask Extension Bug Sl Tp Percentages Change When Changing L

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem may notice that their Stop-Loss (SL) and Take-Profit (TP) percentages change unexpectedly when they adjust the leverage on their trades, leading to unintended trade executions or failed orders.

Technical Root Cause

  • The issue arises from a bug in the Metamask extension's handling of leverage adjustments, which inadvertently alters the SL and TP percentages set by the user, causing discrepancies between the intended and actual trade parameters.

Manual Overrides

  • To temporarily resolve the issue, users can attempt the following manual overrides:
  • Manually re-enter the desired SL and TP percentages after adjusting the leverage.
  • Utilize a third-party trading script to automate trade executions with predefined parameters.
  • Edit the trade configuration file directly using the following code:
{
  "tradeConfig": {
    "leverage": 5,
    "stopLoss": 0.05,
    "takeProfit": 0.10
  }
}

Consider utilizing an external console for more complex trade configurations.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum (ETH) wallet connected
  • Basic understanding of trading terminology and risk management principles
