Metamask Extension Bug Sl Tp Percentages Change When Changing L

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem may notice that their Stop Loss (SL) and Take Profit (TP) percentages change unexpectedly when they adjust the leverage of their positions.

Technical Root Cause

  • The issue arises from a bug within the Metamask extension that incorrectly recalculates SL and TP percentages based on leverage changes, rather than maintaining the originally set values.

Manual Overrides

  • To temporarily resolve the issue, users can attempt the following manual overrides:
  • Set SL and TP percentages manually after each leverage adjustment.
  • Use a third-party calculator to determine the correct SL and TP values based on the desired leverage.
  • Implement a custom script to automatically adjust SL and TP percentages according to the user's predefined strategy:
{
  "leverage": 2,
  "SL_percent": 5,
  "TP_percent": 10
}

Consider using an external console for more complex scripting needs.

Prerequisites

  • Metamask extension installed and configured.
  • Basic understanding of leverage and SL/TP strategies.
  • Familiarity with custom scripting languages (e.g., JavaScript).
