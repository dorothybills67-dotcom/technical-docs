Metamask Extension Bug Sl Tp Percentages Change When Changing L

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem experience a bug where Stop-Loss (SL) and Take-Profit (TP) percentages change unexpectedly when adjusting the leverage level.

Technical Root Cause

  • The issue arises from a mismatch in the extension's internal state management, where the SL and TP percentages are recalculated based on the new leverage level, causing unintended changes to the user's predefined settings.

Manual Overrides

  • To manually override the bug, users can attempt the following:
  • Update the Metamask extension to the latest version
  • Clear the browser's cache and storage for the extension
  • Use the following JSON configuration to reset the SL and TP percentages:
{
  "slPercentage": 5,
  "tpPercentage": 10,
  "leverage": 2
}

Consider utilizing an external console for more advanced configuration and debugging.
