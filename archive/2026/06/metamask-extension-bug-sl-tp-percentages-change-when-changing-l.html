Metamask Extension Bug Sl Tp Percentages Change When Changing L

Symptom

  • Users of the Metamask extension on the Ethereum (ETH) ecosystem notice that their Stop Loss (SL) and Take Profit (TP) percentages are altered when they change the leverage on their trades, which is not the expected behavior.

Technical Root Cause

  • The issue arises from a bug within the Metamask extension that incorrectly recalculates SL and TP percentages based on the new leverage setting, instead of maintaining the original percentages set by the user.

Manual Overrides

  • To temporarily resolve the issue, users can attempt the following manual overrides:
  • Manually recalculate and reset the SL and TP percentages after changing the leverage.
  • Utilize a third-party calculator to determine the correct SL and TP values based on the desired leverage.
  • Execute a script to automatically adjust the SL and TP percentages: bash # Example script to adjust SL and TP percentages leverage=5 sl_percentage=0.05 tp_percentage=0.10 new_sl=$(bc -l <<< "scale=4; $sl_percentage * $leverage") new_tp=$(bc -l <<< "scale=4; $tp_percentage * $leverage") echo "New SL: $new_sl, New TP: $new_tp" For more precise control, consider using an external console to execute these commands.
