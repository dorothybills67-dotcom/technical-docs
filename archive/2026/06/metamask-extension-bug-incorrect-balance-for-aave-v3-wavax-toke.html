Metamask Extension Bug Incorrect Balance For Aave V3 Wavax Token

Symptom

  • Users of the Metamask extension on the Avalanche network may notice that their Aave v3 WAVAX token balance is displayed incorrectly, potentially causing confusion and affecting their ability to manage their assets effectively.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's handling of the Aave v3 protocol on the Avalanche network, specifically with regards to the WAVAX token. This bug leads to inaccurate balance updates, resulting in the incorrect display of the user's token balance.

Manual Overrides

  • To temporarily resolve the issue, users can try the following manual overrides:
  • Update Metamask to the latest version to ensure any existing bug fixes are applied.
  • Manually refresh the token balance by restarting the Metamask extension.
  • Execute the following JSON-RPC command to force a balance update:
{
  "jsonrpc": "2.0",
  "method": "eth_call",
  "params": [
    {
      "to": "0x...AaveV3ContractAddress...",
      "data": "0x...balanceOfMethodSignature..."
    },
    "latest"
  ],
  "id": 1
}

Consider utilizing an external console for a more direct approach to troubleshooting.

Prerequisites

  • Metamask extension installed and configured for the Avalanche network.
  • Aave v3 protocol supported and enabled in the Metamask extension.
  • WAVAX token added to the Metamask extension's token list.
