Metamask Extension Bug Account Discovery And Syncs Error For Social Login Import Users

Symptom

  • Users encounter an error when attempting to import their accounts using social login, resulting in failed account discovery and syncs.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, specifically with the social login import feature, which fails to properly handle account discovery and synchronization.

Manual Overrides

  • Attempt to manually reset the Metamask extension by deleting local storage: localStorage.clear()
  • Use the Metamask API to manually sync accounts: 
{
  "method": "eth_accounts",
  "params": [],
  "id": 1,
  "jsonrpc": "2.0"
}
  • Try disabling and re-enabling the social login import feature in the Metamask settings.

Prerequisites: * Metamask extension installed and enabled * Social login feature enabled in Metamask settings * Ethereum network selected in Metamask

For further troubleshooting, consider using an external console to execute manual override commands.
