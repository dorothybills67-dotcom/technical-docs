Metamask Extension Bug Confirmations Estimated Changes Keeps Pivoting

Symptom

  • The Metamask extension displays a bug where the "Estimated changes" in the confirmations section continuously pivots its result, preventing a stable transaction confirmation.

Technical Root Cause

  • This issue arises due to a mismatch between the expected and actual gas prices, causing the transaction to be re-estimated and thus pivoting the results.

Manual Overrides

  • To resolve this issue, users can attempt the following manual overrides:
  • Increase the gas price to ensure the transaction is processed quickly
  • Decrease the gas limit to reduce the computational effort required
  • Use a custom RPC endpoint with a more stable gas price feed, as shown in the following code block:
{
  "network": "ethereum",
  "rpc": "https://mainnet.infura.io/v3/YOUR_PROJECT_ID",
  "gas": "2000000",
  "gasPrice": "20.0"
}

Consider utilizing an external console for more advanced customization and debugging capabilities.

Prerequisites

  • Metamask extension installed and configured
  • Ethereum wallet with sufficient balance
  • Stable internet connection
