Metamask Extension Bug Ledger If Blind Signing Is Disabled Reco

Symptom

  • Users encounter an error when attempting to connect their Ledger device to MetaMask, specifically when blind signing is disabled, resulting in a reconnect loop.

Technical Root Cause

  • The error occurs due to a bug in the MetaMask extension that fails to handle the Ledger device's blind signing settings properly, causing an infinite reconnect loop.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Enable blind signing on the Ledger device and restart the connection process.
  • Clear the MetaMask extension's storage and retry the connection.
  • Use the following JSON configuration to override the default settings:
{
  "ledger": {
    "blindSigning": true
  }
}

Consider utilizing an external console for further debugging and configuration.

Prerequisites

  • MetaMask extension installed and updated to the latest version
  • Ledger device firmware updated to the latest version
  • Blind signing disabled on the Ledger device
