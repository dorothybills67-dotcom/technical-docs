Metamask Extension Bug Ledger If Blind Signing Is Disabled Reco

Symptom

  • Users encounter an error when attempting to connect their Ledger device to Metamask, specifically after disabling blind signing, resulting in a prompt to reconnect from the beginning of the workflow.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension that fails to handle the scenario where blind signing is disabled on the Ledger device, leading to a connection issue.

Manual Overrides

  • To resolve the issue, try the following manual overrides:
  • Enable blind signing on the Ledger device and then disable it again to restart the connection process.
  • Update the Metamask extension to the latest version to ensure any bug fixes are applied.
  • Use the following JSON configuration to manually configure the connection settings:
{
  "ledger": {
    "blindSigning": true
  }
}

Consider using an external console to execute these commands for a more streamlined experience.

Prerequisites

  • Metamask extension installed and configured
  • Ledger device connected and set up
  • Blind signing disabled on the Ledger device
