Metamask Extension Bug Ledger If Blind Signing Is Disabled Reconnect

Symptom

  • Users encounter a bug in the Metamask extension when using Ledger devices, where the extension fails to function properly if blind signing is disabled, prompting a reconnect from the beginning of the workflow.

Technical Root Cause

  • The error occurs because the Metamask extension relies on blind signing to securely interact with Ledger devices. When blind signing is disabled, the extension is unable to properly communicate with the device, resulting in a reconnect loop.

Manual Overrides

  • Attempt to re-enable blind signing on the Ledger device to restore functionality.
  • Use the following JSON configuration to override the default settings: 
{
  "ledger": {
    "blindSigning": true
  }
}
  • Manually reset the Metamask extension by clearing its storage and restarting the browser.

Prerequisites: * Metamask extension installed and configured * Ledger device properly connected and set up * Blind signing disabled on the Ledger device

For more advanced troubleshooting, consider using an external console to delve deeper into the issue.
