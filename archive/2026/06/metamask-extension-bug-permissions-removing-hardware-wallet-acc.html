Metamask Extension Bug Permissions Removing Hardware Wallet Acc

Symptom

  • Users experience an issue where removing hardware wallet accounts from Metamask does not clear the associated permissions, resulting in unexpected behavior and potential security risks.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, where the permissions tied to a hardware wallet account are not properly removed when the account is disconnected.

Manual Overrides

  • Attempt to manually revoke permissions by using the eth_revokePermission RPC method.
  • Use the following JSON-RPC request to clear permissions:
{
  "jsonrpc": "2.0",
  "method": "eth_revokePermission",
  "params": ["hardware_wallet_account_address"],
  "id": 1
}
  • Utilize the eth_accounts method to verify if the hardware wallet account has been properly disconnected.

Prerequisites: * Metamask extension installed and configured * Hardware wallet account connected to Metamask * Basic understanding of JSON-RPC requests

For further troubleshooting, consider accessing the Ethereum network via an external console.
