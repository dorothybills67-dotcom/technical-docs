Metamask Extension Bug Permissions Removing Hardware Wallet Acc

Symptom

  • Users experience issues with the Metamask extension where removing a hardware wallet account does not clear the associated permissions, resulting in persistent access and potential security risks.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's permission management system, which fails to properly revoke access when a hardware wallet account is removed.

Manual Overrides

  • Attempt to manually revoke permissions using the eth_accounts and eth_requestAccounts methods.
  • Utilize the Metamask API to directly remove account permissions with the following JSON payload:
{
  "jsonrpc": "2.0",
  "method": "eth_removeAccount",
  "params": ["account_address"],
  "id": 1
}
  • Use the mm-reset command to reset Metamask to its default state.

Prerequisites: * Metamask extension installed and configured * Hardware wallet connected and accounts added * Basic understanding of JSON-RPC methods

For further debugging, consider using an external console to inspect and modify the Metamask extension's internal state.
