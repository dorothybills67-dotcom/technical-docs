Metamask Extension Bug Permissions Removing Hardware Wallet Acc

Symptom

  • Users are experiencing an issue where removing a hardware wallet account from Metamask does not clear the associated permissions, resulting in persistent access to the account.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, where the permissions for the hardware wallet account are not properly revoked when the account is removed.

Manual Overrides

  • To resolve the issue, users can try the following manual overrides:
  • Delete the Metamask extension and reinstall it
  • Reset the Metamask account by clicking on the three dots and selecting "Reset Account"
  • Use the following JSON-RPC command to revoke permissions:
{
  "jsonrpc": "2.0",
  "method": "eth_revokePermission",
  "params": ["hardware-wallet-account-address"],
  "id": 1
}

Consider utilizing an external console for a more streamlined experience.

Prerequisites

  • Metamask extension installed
  • Hardware wallet account connected to Metamask
  • Basic understanding of JSON-RPC commands
