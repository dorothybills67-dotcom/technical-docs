Metamask Extension Bug Permissions Removing Hardware Wallet Acc

Symptom

  • Users experience an issue where removing a hardware wallet account from Metamask does not clear the associated permissions, resulting in unexpected behavior and potential security risks.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, where the removal of a hardware wallet account does not properly revoke the previously granted permissions, leading to a mismatch between the user's intended access and the actual permissions stored.

Manual Overrides

  • Attempt to manually revoke permissions by using the eth_revokeAccess RPC method.
  • Use the Metamask API to remove the hardware wallet account and then restart the browser.
  • Execute the following command to clear the permissions cache:
mm-cache-clear --permissions

Consider utilizing an external console for further debugging and management of wallet permissions.

Prerequisites

  • Metamask extension installed and configured
  • Hardware wallet connected and set up
  • Basic understanding of Ethereum RPC methods and wallet management
