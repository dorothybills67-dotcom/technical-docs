Metamask Extension Bug Permissions Removing Hardware Wallet Acc

Symptom

  • Users experience an issue where removing a hardware wallet account from Metamask does not clear the associated permissions, resulting in unexpected behavior and potential security risks.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, where the permissions are not properly updated when a hardware wallet account is removed, leading to a discrepancy between the expected and actual permissions.

Manual Overrides

  • Remove the problematic account and restart the browser: metamask-account-remove --force
  • Clear the Metamask storage to reset permissions: 
{
  "permissions": {},
  "accounts": []
}
  • Manually update the permissions by editing the Metamask configuration file

Consider using an external console to execute these commands for more flexibility.

Prerequisites

  • Metamask extension installed and configured
  • Hardware wallet account connected to Metamask
  • Basic understanding of command-line interfaces and JSON configuration files
