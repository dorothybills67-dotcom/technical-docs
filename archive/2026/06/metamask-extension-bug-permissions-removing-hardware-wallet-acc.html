Metamask Extension Bug Permissions Removing Hardware Wallet Acc

Symptom

  • Users are experiencing an issue where removing a hardware wallet account from the Metamask extension does not clear the corresponding permissions, resulting in inconsistent and potentially insecure access to their Ethereum accounts.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension's permission management system, which fails to properly revoke permissions when a hardware wallet account is removed.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Delete the permissions.json file in the Metamask extension's storage directory
  • Run the following command in the terminal to reset the Metamask extension's permissions:
mm-extension --reset-permissions
  • Manually edit the permissions.json file to remove the offending permissions entries Consider using an external console to verify the changes.

Prerequisites

  • Metamask extension installed and configured
  • Hardware wallet account connected to the Metamask extension
  • Basic understanding of Ethereum and wallet management concepts
