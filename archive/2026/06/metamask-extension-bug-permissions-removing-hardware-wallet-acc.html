Metamask Extension Bug Permissions Removing Hardware Wallet Acc

Symptom

  • Users experience an issue where removing a hardware wallet account from Metamask does not clear the associated permissions, resulting in persistent access and potential security risks.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, where the removal of a hardware wallet account does not properly revoke the previously granted permissions, leading to an inconsistent state.

Manual Overrides

  • Attempt to manually revoke permissions using the eth_accounts and eth_requestAccounts methods.
  • Use the Metamask API to remove the account and then restart the extension.
  • Utilize the following JSON configuration to reset the permissions:
{
  "permissions": {
    "hardwareWallets": []
  }
}

Consider utilizing an external console to execute these commands for a more streamlined experience.

Prerequisites

  • Metamask extension installed and configured
  • Hardware wallet account previously added to Metamask
  • Basic understanding of JSON configuration and Metamask API
