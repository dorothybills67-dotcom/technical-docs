Metamask Extension Bug Permissions Removing Hardware Wallet Acc

Symptom

  • Users encounter an issue where removing a hardware wallet account from Metamask does not clear the associated permissions, leading to unexpected behavior and potential security vulnerabilities.

Technical Root Cause

  • The error occurs due to a bug in the Metamask extension, where the permissions tied to a hardware wallet account are not properly revoked when the account is removed.

Manual Overrides

  • To resolve the issue, users can attempt the following manual overrides:
  • Delete the local storage data associated with the Metamask extension
  • Reset the Metamask extension to its default settings
  • Execute the following command to revoke permissions:
mm-cli --revoke-permissions --account=<hardware-wallet-account-address>

Consider utilizing an external console for enhanced debugging capabilities.

Prerequisites

  • Metamask extension installed and configured
  • Hardware wallet account connected to Metamask
  • Basic understanding of command-line interfaces and wallet management
