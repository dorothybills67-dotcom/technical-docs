Metamask Extension Bug Activity Contract Deployments Are Displayed

Symptom

  • Users notice that contract deployments are incorrectly displayed as contract interactions within the Metamask extension, leading to confusion and potential errors in tracking and managing on-chain activities.

Technical Root Cause

  • The issue arises from a bug in the Metamask extension where the classification of on-chain activities is not properly handled, resulting in the mislabeling of contract deployments as interactions.

Manual Overrides

  • To temporarily resolve the issue, users can attempt the following manual overrides:
  • Check for updates to the Metamask extension to ensure the latest version is installed.
  • Clear the browser cache and restart the browser to refresh the extension's state.
  • Utilize the following JSON configuration to manually correct the activity classification:
{
  "activityClassification": {
    "contractDeployments": "deployments",
    "contractInteractions": "interactions"
  }
}

Consider utilizing an external console for further troubleshooting and configuration adjustments.

Prerequisites

  • Metamask extension installed and configured.
  • Basic understanding of Ethereum blockchain and smart contract interactions.
  • Familiarity with JSON configuration files and manual override techniques.
