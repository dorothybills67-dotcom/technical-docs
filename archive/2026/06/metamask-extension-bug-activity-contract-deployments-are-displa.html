Metamask Extension Bug Activity Contract Deployments Are Displayed as Contract Interaction

Symptom

  • Users of the Metamask extension may notice that contract deployments are incorrectly displayed as contract interactions within the activity tab, leading to confusion and potential mismanagement of on-chain activities.

Technical Root Cause

  • This issue arises from a bug in the Metamask extension's logic for categorizing and displaying different types of on-chain activities, specifically failing to differentiate between contract deployments and interactions.

Manual Overrides

  • To temporarily resolve this issue, users can attempt the following manual overrides:
  • Clear the Metamask extension's local storage to reset its state.
  • Manually filter activities by transaction type using external blockchain explorers.
  • Utilize the following JSON configuration to update the Metamask extension's settings:
{
  "activityFilter": {
    "contractDeployments": true,
    "contractInteractions": false
  }
}

Consider utilizing an external console for more complex configuration adjustments.

Prerequisites

  • Metamask extension installed and configured.
  • Basic understanding of Ethereum blockchain transactions.
  • Access to external blockchain explorers for manual transaction filtering.
